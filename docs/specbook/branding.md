# 스펙북 — 명칭 & 마이크로카피

> 작성일: 2026-06-21
> 범위: 명칭 선택 근거, 카테고리·아이콘 톤, 마이크로카피 표준

---

## 1. 명칭 결정 — "스펙북"

### 1-1. 결정

**확정 명칭: 스펙북 (SpecBook)**

### 1-2. 결정 근거

| 기준 | 평가 |
|---|---|
| 사장님이 한 번에 알아듣는가 | "스펙"은 인테리어 업계에서 자재 사양을 가리키는 이미 익숙한 단어. "북"은 묶음·카탈로그의 시각 메타포. → **들으면 바로 감이 옴.** |
| 짧고 부르기 쉬운가 | 3음절. 카톡·문서 안에서 자주 호출되는 단어이므로 짧을수록 좋음. |
| 견적코치·인테리어코치 톤과 어울리는가 | 스몰테이블 제품군은 "OO코치"가 기본. 스펙북은 다른 음절 패턴이라 **모듈명**으로 구별됨. (코치 = 제품, 스펙북 = 그 안의 도구) |
| 카탈로그/셀렉션 같은 일반 단어보다 구체성 있는가 | "카탈로그"는 자재사 카탈로그와 혼동, "셀렉션"은 영어 잡음. 스펙북은 **현장 단위로 만드는 자료집**이라는 뉘앙스가 명확. |

### 1-3. 대체 후보 비교

| 후보 | 장점 | 단점 | 채택 가능성 |
|---|---|---|---|
| **스펙북** ✅ | 업계 익숙, 짧음, 모듈명 적합 | 영어 어원 (그러나 이미 한국에서 일상어) | 채택 |
| 마감재보드 | 한국어 100%, 직관적 | 4음절, 부르기 무거움, "보드"가 칸반·디자인툴 연상 | 후보 2 |
| 자재선택 | 의미 직설 | 너무 일반적, 다른 모듈명과 충돌 가능 | 후보 3 |
| 셀렉트 / 마감셀렉트 | 짧음 | 영어 잡음, 사장님 톤과 거리 | 탈락 |
| 카탈로그 | 익숙 | 자재사 PDF 카탈로그와 혼동 | 탈락 |
| 픽북(PickBook) | 짧음 | "픽"이 캐주얼·테크 톤. 인테리어 현장 톤 아님 | 탈락 |

### 1-4. 사용 규칙

- **공식 표기**: 스펙북 (한글)
- **영문 병기 필요 시**: SpecBook (대소문자 고정, 한 단어로 표기 — Spec book 금지)
- **모듈 호출**: "스펙북", "스펙북 만들기", "이 현장의 스펙북"
- **부르지 않을 표현**: "스펙북 솔루션", "스펙북 시스템" — 모듈 자체가 도구이므로 수식어 불필요

## 2. 카테고리 라벨 & 아이콘 톤

### 2-1. 공종 라벨 (스펙북 기본 12분류)

| 공종 | 표시 라벨 | 아이콘(lucide) | 비고 |
|---|---|---|---|
| 바닥재 | 바닥 | `Layers` | "마루"로 좁히지 않고 통합 |
| 도배 | 도배 | `Brush` | 벽지·실크 모두 포함 |
| 타일 | 타일 | `Grid3x3` | 욕실·주방·현관 묶음 |
| 주방 | 주방 | `ChefHat` | 싱크대·상판·후드 묶음 |
| 욕실 | 욕실 | `Bath` | 양변기·세면대·수전 묶음 |
| 조명 | 조명 | `Lamp` | 다운라이트·팬던트 묶음 |
| 도어 | 도어 | `DoorOpen` | 현관·방문 |
| 중문 | 중문 | `DoorClosed` | 도어와 시각적으로 구분 |
| 샷시 | 샷시 | `Maximize2` | 시스템창·일반창 묶음 |
| 스위치·콘센트 | 스위치 | `Lightbulb` | 짧게 줄여 표기 |
| 가구 | 가구 | `Sofa` | 붙박이장·신발장 |
| 필름 | 필름 | `Sticker` | 도어·몰딩 필름 |

- 아이콘 컬러: 단일 톤(`var(--muted)`). 공종이 활성/완료일 때만 `var(--green)`으로 변경.
- 아이콘 사이즈: 14~16px (목록), 24px (카드 헤더), 32px (빈 상태 일러).
- 이모지 아이콘은 **사용 금지** (브랜드 v2 §5 정책 준수). 단 마케팅 카피·소셜 포스트에서는 1~2개 허용.

### 2-2. 컬러 의미

| 상태 | 시각 표시 | 의미 |
|---|---|---|
| 미정 | 회색 점 + "미정" | 고객 아직 선택 안 함 |
| 검토중 | 주황 점 + "검토중" | 고객이 봤지만 선택 보류 |
| 제출 완료 | `var(--green)` 체크 + "제출 완료" | 고객 결정, 사장님 확인 대기 |
| 사장님 확정 | 초록 채워진 체크 + "확정" | 사장님이 최종 승인. 발주 가능. |
| 변경 요청 | 빨간 점 + "변경" | 고객/사장님 중 누군가 다시 골라달라 요청. |

> "거절" 같은 단어는 쓰지 않음. "변경 요청"으로 부드럽게.

## 3. 스몰테이블 제품군 톤 일관성

세 제품의 톤 차이를 미세하게 분리해 일관성 유지.

| 제품 | 대상 | 톤 |
|---|---|---|
| **스몰테이블** | 일반 소비자 (집주인) | 따뜻함·감성·현장 사진 중심 |
| **견적코치** | 소비자 + 일부 업체 | 친절한 설명·숫자 근거·"적정가" |
| **인테리어코치** (← 스펙북 위치) | 인테리어 업체 사장님 | 현장 점퍼 톤·실무 언어·"사장님 호칭"·과장 없음 |

### 3-1. 스펙북 카피 톤 — 한 줄로

> **"엑셀 들고 카톡으로 자재 정하시던 사장님께, 한 화면으로 옮겨드리는 도구"** 같은 자세. 위에서 가르치지 않고, 옆에서 같이 일해본 사람처럼.

## 4. UI 마이크로카피 표준

### 4-1. 빈 상태 (Empty States)

**스펙북 목록 — 처음 진입했을 때**
```
아직 만든 스펙북이 없어요.

자주 쓰는 평형·타입별로 한 번 만들어 두면,
다음 현장은 복제해서 1분 안에 시작할 수 있습니다.

[ 첫 스펙북 만들기 ]   [ 공용 템플릿 보기 ]
```

**공종 안에 아이템 없음**
```
아직 옵션을 추가하지 않으셨어요.
자주 쓰시는 자재 2~3개를 골라 담아주세요.

[ 자재 추가 ]
```

**고객 측 — 카탈로그 처음 열었을 때 (옵션 비어있음)**
```
사장님이 아직 옵션을 준비 중이세요.
잠시 후 다시 열어 주세요.
```

### 4-2. 공유 링크 발급/전달 안내

**링크 발급 직후 토스트**
```
링크가 생성됐어요. 카톡으로 보내실 수 있습니다.
```

**링크 복사 버튼 옆 안내**
```
이 링크는 이 현장 전용입니다. 다른 현장과 섞이지 않아요.
```

**링크 만료/재발급 안내**
```
링크를 새로 발급하면 기존 링크는 그 시점부터 안 열려요.
이미 고객이 제출한 내용은 그대로 남습니다.
```

### 4-3. 고객 제출 완료 문구

**고객 측 — 모든 공종 제출 완료**
```
선택 결과를 사장님께 전달했어요.

사장님이 확인 후 다시 연락드릴 거예요.
변경하실 일이 생기면 이 링크를 다시 열어 알려주시면 됩니다.
```

**사장님 측 — 신규 제출 알림**
```
[현장명] 고객님이 마감재 선택을 보내셨어요.
17개 항목 중 15개 제출 · 2개 미정.
```

### 4-4. 사장님 확정 / 변경 요청

**사장님 → "확정" 버튼 옆 안내**
```
확정하시면 발주서로 바로 옮길 수 있어요.
이후 변경은 고객 동의가 필요합니다.
```

**사장님 → "변경 요청" 버튼 옆**
```
고객님이 카탈로그에서 다시 고르시도록 안내드립니다.
이미 골라두신 다른 항목은 그대로 남아 있어요.
```

### 4-5. 알림 문구 (인앱 / SMS)

**인앱 알림 — 고객 제출**
```
[잠실 32평] 카탈로그 제출이 도착했습니다.
```

**SMS — 고객 미응답 D+3 (선택 발송)**
```
[인테리어코치] {업체명}이(가) 보내드린 마감재 선택 링크를
아직 못 보셨어요. 결정에 시간이 더 필요하시면 알려주세요.
```

**SMS — 사장님이 확정 후 고객에게**
```
[{업체명}] 골라주신 마감재로 발주 들어갑니다. 감사합니다.
```

### 4-6. 에러 / 실패 문구

| 상황 | 카피 |
|---|---|
| 링크 만료된 카탈로그 열기 | "이 링크는 만료됐어요. 사장님께 새 링크를 요청해 주세요." |
| 네트워크 오류 저장 실패 | "저장이 안 됐어요. 잠시 후 다시 시도해 주세요." |
| 권한 없음 (다른 워크스페이스) | "이 카탈로그에는 접근 권한이 없어요." |
| 자재 옵션 0개로 발행 시도 | "공종에 옵션이 없으면 고객이 선택할 게 없어요. 옵션을 먼저 추가해 주세요." |

### 4-7. 헬프 텍스트 (인풋 보조)

| 인풋 | 헬프 |
|---|---|
| 스펙북 이름 | "현장명으로 자동 채워집니다. 바꿔도 됩니다." |
| 공종 추가 | "쓰지 않는 공종은 비워두셔도 됩니다." |
| 자재 옵션 | "한 공종에 2~3개를 추천드려요. 너무 많으면 고객이 결정 못 합니다." |
| 메모 | "고객한테 보일 코멘트. 예: '같은 톤이라 안 보이지만 디테일이 달라요'" |

## 5. 카피 체크리스트 (배포 전)

스펙북 관련 카피·UI 변경 배포 전 1분 점검.

- [ ] "사장님" 호칭 사용, "고객님/귀하" 미사용
- [ ] 영어 마케팅 용어(셀렉션·인사이트·옵티마이즈 등) 없음
- [ ] 과장 표현(혁신적·완벽한·100%) 없음
- [ ] 이모지 — UI 버튼·인풋에는 없음, 헤더/카드/마케팅 카피에만 사용
- [ ] 숫자 — 검증 가능한 값만 (868건 자재 DB, 17개 항목 등)
- [ ] "올인원 SaaS" 단어 사용 안 함
- [ ] 액션 결과를 미리 알려주는 문구 (예: "확정하시면 발주서로 옮겨집니다")
