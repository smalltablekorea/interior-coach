## 1. 핵심 원칙

> "전문가스럽지 않되 신뢰감 있게, 현장 감각 + 테크 감성 공존."

- **말투**: 해요체 기반 존댓말. 사무실 정장 톤 ❌ · 현장 점퍼 톤 ⭕. 단, 테크 제품이라는 정체성은 유지(숫자·모듈·자동화 용어 OK).
- **출발점**: 사장님이 "이 사람들 현장 아는 사람이네"라고 느끼게. 업계 용어(공종·기성·미수금·반장)는 자연스럽게 써도 되지만, 마케팅 영어(페르소나·핏·밸류·퍼널·리텐션)는 금지.
- **증명 방식**: 수식어가 아니라 **숫자와 상황**. "혁신적인 AI" ❌ · "영수증 92~96% 자동 인식" ⭕.

## 2. 금지 / 권장 표현 각 10

### 금지 표현 10 (마케팅 과장 + 영어 잡음)

| # | 금지 | 왜 금지 |
|---|---|---|
| 1 | 혁신적인 / 압도적인 / 차별화된 | 구체 근거가 없는 수식어. 수치로 바꿔 쓸 것. |
| 2 | 올인원 솔루션 / 통합 플랫폼 | "플랫폼·솔루션"은 빈 단어. 기능을 직접 말할 것. |
| 3 | 디지털 전환 / 업무 혁신 | 사장님 말투 아님. "카톡·엑셀 안 엽니다"처럼 치환. |
| 4 | 페르소나 / 밸류 / 핏 / 퍼널 | 마케팅 영어. 사장님 카피엔 금지. |
| 5 | 고객 경험 / CX | 추상. "고객한테 전화 덜 옵니다"로. |
| 6 | 100% 자동화 / 0원 비용 / 완벽한 | 과장. 법적·품질 리스크. |
| 7 | 업계 1위 / 국내 최초 / 최첨단 | 검증 부담. "자재 DB 868건"처럼 숫자로. |
| 8 | 귀하 / 귀사 / 고객님 | 너무 고객센터 말투. "사장님 / 대표님"이 더 자연스러움. |
| 9 | ~하여 / ~함으로써 / ~임에도 불구하고 | 글말투. 현장 말투가 아님. |
| 10 | 이모지 남용 (🔥🚀💪✨) 한 문장 3개 이상 | 유치·산만. §5 이모지 정책 참조. |

### 권장 표현 10

| # | 권장 | 사용 예 |
|---|---|---|
| 1 | 사장님 / 대표님 | "사장님이 안 봐도 장부가 써집니다." |
| 2 | 현장·기성·미수금·반장 | 업계 용어 자연스럽게 노출. 신뢰감 상승. |
| 3 | 반나절이 3분 / 전화 4번이 0번 | 비교 수치. |
| 4 | 카톡 / 엑셀 / 영수증 / 통장 | 사장님 현실 도구를 언급. |
| 5 | 찍으면 / 지나가면 / 사진만 / 한 번 | 사장님 실제 동작. |
| 6 | 한 라인·한 줄·한 화면 | 올인원을 구체 이미지로. |
| 7 | OCR·AI가 먼저 읽습니다 | 자동화를 인격화. |
| 8 | 868건 / 92~96% / 3분 / 14일 | 검증 가능한 숫자. |
| 9 | 현직 대표가 매일 씁니다 | "만든 사람"이 아니라 "쓰는 사람". |
| 10 | ~됩니다 / ~씁니다 / ~보입니다 | 결과 서술형. 행동을 지시하지 않음. |

## 3. 1인칭 원칙

- 브랜드 주체는 **"인테리어코치"** 로 통일. "저희는·우리는" 사용 최소화.
  - ✅ "인테리어코치는 영수증을 먼저 읽습니다."
  - ❌ "저희 인테리어코치는 고객님의 영수증을..."
- 단, Founder Story 같은 1인칭 맥락에서는 **"배다솜·스몰테이블 대표"** 명시 후 "저"로 시작 가능.
  - ✅ "저는 잠실 르엘 현장에서 이 제품으로 일합니다."
- 고객은 **"사장님·대표님"**. "고객님·귀하·유저·이용자"는 금지.
- "우리" 는 타겟 사장님과 "같은 편"이라는 맥락에서만 사용.
  - ✅ "우리 업계, 영수증 한 번도 안 밀려본 대표 드뭅니다."

## 4. 숫자 표기 규칙

| 상황 | 표기 |
|---|---|
| 자재 DB | **868건** (고정) |
| 동시 현장 | **3현장 / 5현장** (복수 형태 우선) |
| 영수증 인식률 | **92~96%** (레인지로 표기, "100%" 금지) |
| 가격 | **₩79,000 / 월** · **₩790,000 / 년** (천 단위 쉼표 + 원화 기호) |
| 연 할인 | **17%** (소수점 없이 반올림) |
| 팀원 | **3명 / 5명** (숫자 + 명) |
| 공종 | **12공종 · 14공종** 등 숫자 + 공종 |
| 기간 | **14일 / 3분 / 6주** (숫자 + 한글 단위) |
| 시간대 | **밤 11시 / 월말 / 4월 16일** (한글) |

- 숫자는 **타불러 숫자(Tabular Nums)** 권장 (가격·통계 카드).
- 외래어 숫자 단위(M·K·B) 금지. 한글 단위 사용.

## 6. 폰트 (v1 대비 reality 업데이트)

- `src/app/layout.tsx` 에서 실제 로드: **Geist Sans + Geist Mono** (Google Fonts via `next/font/google`)

### 라이트 테마

`[data-theme="light"]` 블록(`globals.css:22~`)에 별도 정의. `--green` = `#00A85E`(다크보다 조금 더 진함, 흰 배경 대비).

### v1 대비 변경 사항

- v1 명세 `--background #0A0A0A` → 실제 `#050505`. **코드 값으로 통일**.
- v1 명세 `--blue #3B82F6` → 실제 `#4A9EFF`. **코드 값으로 통일**.
- v1 명세 `--amber #F59E0B` → 코드에는 없음. `--orange #FF9F43` 만 존재. **"amber" 토큰 폐기, orange로 일원화**.
- 컬러 사용 비율은 v1 권고 유지: Green 20% / Dark 60% / White 15% / Accent 5%.

## 10. 체크리스트 (카피 배포 전)

- [ ] "현장 톡방 / 고객 포털 / 전자서명" 언급 없음
- [ ] "100% / 0원 / 완벽한 / 업계 1위" 같은 과장 없음
- [ ] 숫자는 모두 검증 가능한 근거 있음 (868건·92~96%·17%)
- [ ] 이모지가 §5 정책 범위 내
- [ ] "귀하·고객님·유저" → "사장님·대표님"으로 치환
- [ ] 영어 마케팅 용어(페르소나·핏·밸류) 0건
- [ ] 가격 표기 천 단위 쉼표 포함 (`₩79,000`)
- [ ] 코드 컬러 토큰(`var(--green)` 등) 참조, HEX 직접 쓰지 않음
