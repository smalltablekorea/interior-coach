# 견적코치 마케팅 센터 — 운영 전략 요약

> 최종 수정: 2026-03-21
> 대상: admin 마케팅 센터 구현 팀 (프론트엔드, 백엔드, 운영)

---

## 1. 마케팅 센터란?

견적코치 서비스의 **운영형 마케팅 OS**.
단순 트래픽 증가가 아니라, **로그인한 고의도 사용자**를 퍼널 단계별로 전환·재방문시키는 것이 목적이다.

### 핵심 퍼널

```
방문 → 회원가입 → 로그인 → 업로드 시작 → 업로드 제출
→ 분석 완료 → 결제 시작 → 결제 성공 → 리포트 확인 → 업체문의
```

### 운영 원칙

| 원칙 | 설명 |
|------|------|
| 로그인 강제 유지 | 장난/스팸 업로드 방지. 비회원 업로드 절대 허용하지 않음 |
| 고의도 사용자 중심 | 실제 인테리어 견적 비교가 필요한 사람만 타겟 |
| 민감 데이터 보호 | 업로드 원문 파일, 개인 견적 상세는 마케팅 UI에 노출 금지 |
| 신뢰 기반 커뮤니케이션 | 과장 광고 지양, 객관적 분석·절감 근거 중심 |
| 단계별 전환 최적화 | 각 퍼널 단계 이탈자에 맞춤형 넛지 제공 |

## 2. 메뉴별 목적 정의

### 2-1. 개요 (Overview)

**목적**: 오늘 서비스 상태를 한눈에 파악하고, 즉시 조치가 필요한 사항을 식별한다.

- KPI 카드 8개: 핵심 전환 지표의 현재값 + 전일/전주 대비 변화
- 오늘 액션 필요 카드: 이탈 유저 리스트 (미결제, 업로드 중단, 결제 실패, 휴면)
- 이상징후 카드: 전환율 급락, 결제 실패율 상승, 업로드 실패율 상승 감지

**핵심 질문**: "오늘 서비스에서 가장 먼저 해야 할 일이 뭐지?"

### 2-2. 퍼널 (Funnel)

**목적**: 전체 사용자 흐름을 시각화하고, 병목 지점을 찾아낸다.

- 10단계 퍼널 시각화 (방문 → 업체문의)
- 날짜·채널·캠페인·디바이스·지역별 필터
- 단계별 전환율 + 이탈률 수치 표시
- 이탈이 가장 심한 단계를 자동 하이라이트

**핵심 질문**: "유저가 어디에서 가장 많이 빠지는가?"

### 2-3. 리드 (Leads)

**목적**: 개별 사용자의 행동 상태를 추적하고, 적절한 넛지를 판단한다.

- 사용자 리스트 (행동·세그먼트·점수 기반)
- 리드 상세 패널: 행동 타임라인, 전환 상태, 추천 액션
- 업로드 원문은 노출 금지 — 구조화된 상태(시작/제출/완료)만 표시
- 리드 점수 자동 계산

**핵심 질문**: "이 유저에게 지금 어떤 액션이 필요한가?"

### 2-4. 세그먼트 (Segments)

**목적**: 유사 행동 패턴의 사용자 그룹을 정의하고, 그룹별 마케팅을 운영한다.

- 9개 기본 세그먼트 시드 제공
- 동적 세그먼트 규칙 (JSON/typed config)
- 세그먼트별 유저 수, 최근 변화, 전환율

**핵심 질문**: "어떤 유저 그룹에 집중해야 ROI가 가장 높은가?"

### 2-5. 자동화 (Automations)

**목적**: 반복적인 넛지/리마인더를 자동으로 실행한다.

- 트리거 → 조건 → 지연 → 액션 구조
- 5개 기본 자동화 시드 (가입 후 미업로드, 업로드 미제출, 미결제, 결제 후 유도, 후기 요청)
- 안전장치: 야간 금지, 중복 제한, 최근 결제자 제외

**핵심 질문**: "수동 개입 없이 전환을 올릴 수 있는가?"

### 2-6. 캠페인 (Campaigns)

**목적**: 기간 한정 프로모션이나 광고를 기획·집행·추적한다.

- 캠페인 생성/상세 관리 (초안 → 승인대기 → 집행중 → 종료)
- 채널: 이메일, 카카오, 검색광고, 리타게팅, 콘텐츠
- UTM 자동 생성, 예산/성과 추적

**핵심 질문**: "이번 달 어떤 캠페인을 돌려야 하는가?"

### 2-7. 콘텐츠 스튜디오 (Content Studio)

**목적**: 광고·이메일·카카오·블로그 문구를 견적코치 톤에 맞게 생성·관리한다.

- 카테고리별 템플릿 (검색광고, 이메일, 카카오, 블로그, 랜딩, 후기요청)
- 견적코치 도메인에 맞는 한국어 카피 시드
- 저장/편집/복제 지원

**핵심 질문**: "이 채널에 맞는 카피를 빠르게 만들 수 있는가?"

### 2-8. 실험 (Experiments)

**목적**: A/B 테스트를 통해 전환율 개선 가설을 검증한다.

- 대상: 헤드라인, CTA, 가격 카피, 후기 배치 등
- 상태: 초안 → 실행중 → 종료
- 주요 지표: 클릭률, 업로드 시작률, 결제 시작률, 결제 성공률

**핵심 질문**: "이 변경이 정말 전환에 도움이 되는가?"

### 2-9. 설정 (Settings)

**목적**: 채널 연동, 발송 정책, UTM 규칙, 개인정보 동의 등을 관리한다.

- 채널별 연동 상태 (이메일/카카오/SMS)
- 발송 제한 정책 (야간 금지, 일일 한도 등)
- 기본 UTM 규칙
- 운영자 알림 설정
- 개인정보/마케팅 동의 문구

### 이벤트 공통 필드

```typescript
interface MarketingEvent {
  id: string;
  event_name: string;
  user_id: string | null;        // 로그인 유저
  session_id: string;
  anonymous_id: string | null;   // 비로그인 식별 (쿠키)
  occurred_at: Date;
  properties: Record<string, unknown>;  // 이벤트별 추가 속성
  // UTM 추적
  source: string | null;         // utm_source
  medium: string | null;         // utm_medium
  campaign: string | null;       // utm_campaign
  content: string | null;        // utm_content
  term: string | null;           // utm_term
  // 컨텍스트
  landing_path: string | null;
  referrer: string | null;
  device_type: string | null;    // desktop, mobile, tablet
  browser: string | null;
  os: string | null;
  region: string | null;
  ip_hash: string | null;        // 원문 IP 저장 금지
  // 실험
  experiment_variant: string | null;
}
```

| 회원가입 | +10 | 최초 1회 |
| 로그인 | +5 | 일 1회만 반영 |
| 업로드 시작 | +20 | |
| 업로드 제출 | +30 | 분석 요청 의사 확인 |
| 분석 완료 | +15 | 시스템 이벤트 |

### 리드 등급

| 등급 | 점수 범위 | 의미 |
|------|-----------|------|
| Hot | 80+ | 결제 직전 또는 완료, 즉시 대응 |
| Warm | 40–79 | 활발한 탐색 중, 넛지 효과적 |
| Cold | 0–39 | 가입만 하거나 활동 저조, 리인게이지 대상 |
| Dormant | 음수 | 장기 미접속, 재활성 캠페인 대상 |
