# 견적코치 마케팅 센터 — KPI 정의서

## 1. 퍼널 KPI (개요 대시보드 카드용)

### KPI-01: 신규가입 (New Signups)

| 항목 | 값 |
|------|-----|
| **의미** | 해당 기간에 회원가입을 완료한 사용자 수 |
| **계산식** | `COUNT(signup_completed) WHERE occurred_at IN 기간` |
| **이벤트** | `signup_completed` |
| **표시** | 절대값 (예: "127명") + 전일/전주 대비 증감률 |
| **증감 색상** | 증가=green, 감소=red |
| **갱신 주기** | 실시간 또는 5분 |
| **노출 위치** | 개요 > KPI 카드 1번 |

### KPI-02: 업로드 시작 (Upload Started)

| 항목 | 값 |
|------|-----|
| **의미** | 로그인 후 견적서 업로드를 시작한 사용자 수 |
| **계산식** | `COUNT(DISTINCT user_id) WHERE event = upload_started AND occurred_at IN 기간` |
| **이벤트** | `upload_started` |
| **표시** | 절대값 + 전일/전주 대비 증감률 |
| **세부** | 가입 대비 시작률 = `업로드 시작 / 신규가입 × 100` |
| **노출 위치** | 개요 > KPI 카드 2번 |

### KPI-03: 업로드 제출 (Upload Submitted)

| 항목 | 값 |
|------|-----|
| **의미** | 견적서 파일 업로드를 완료하고 분석을 요청한 사용자 수 |
| **계산식** | `COUNT(DISTINCT user_id) WHERE event = upload_submitted AND occurred_at IN 기간` |
| **이벤트** | `upload_submitted` |
| **표시** | 절대값 + 전일/전주 대비 증감률 |
| **세부** | 시작 대비 제출률 = `업로드 제출 / 업로드 시작 × 100` |
| **노출 위치** | 개요 > KPI 카드 3번 |

### KPI-04: 분석 완료 (Analysis Completed)

| 항목 | 값 |
|------|-----|
| **의미** | AI 견적 분석이 완료된 건수 (시스템 이벤트) |
| **계산식** | `COUNT(analysis_completed) WHERE occurred_at IN 기간` |
| **이벤트** | `analysis_completed` |
| **표시** | 절대값 + 전일/전주 대비 증감률 |
| **주의** | 시스템 처리 시간 이슈로 제출 대비 완료율이 100%가 아닐 수 있음 |
| **노출 위치** | 개요 > KPI 카드 4번 |

### KPI-05: 결제 시작 (Checkout Started)

| 항목 | 값 |
|------|-----|
| **의미** | 결제 화면(paywall)에 진입하거나 결제 수단을 선택한 사용자 수 |
| **계산식** | `COUNT(DISTINCT user_id) WHERE event IN (paywall_viewed, checkout_started) AND occurred_at IN 기간` |
| **이벤트** | `paywall_viewed`, `checkout_started` |
| **표시** | 절대값 + 전일/전주 대비 증감률 |
| **노출 위치** | 개요 > KPI 카드 5번 |

### KPI-06: 결제 성공 (Payment Succeeded)

| 항목 | 값 |
|------|-----|
| **의미** | 결제가 성공적으로 완료된 건수 |
| **계산식** | `COUNT(payment_succeeded) WHERE occurred_at IN 기간` |
| **이벤트** | `payment_succeeded` |
| **표시** | 절대값 + 전일/전주 대비 증감률 |
| **세부** | 결제 전환율 = `결제 성공 / 분석 완료 × 100` |
| **노출 위치** | 개요 > KPI 카드 6번 |

### KPI-07: 매출 (Revenue)

| 항목 | 값 |
|------|-----|
| **의미** | 해당 기간의 총 결제 금액 |
| **계산식** | `SUM(payment_succeeded.properties.amount) WHERE occurred_at IN 기간` |
| **이벤트** | `payment_succeeded` (properties.amount 필드) |
| **표시** | 금액 형식 (예: "₩3,840,000") + 전일/전주 대비 증감률 |
| **노출 위치** | 개요 > KPI 카드 7번 |

### KPI-08: 업체문의 (Inquiry Submitted)

| 항목 | 값 |
|------|-----|
| **의미** | 리포트 확인 후 추천 업체에 문의를 제출한 건수 |
| **계산식** | `COUNT(inquiry_submitted) WHERE occurred_at IN 기간` |
| **이벤트** | `inquiry_submitted` |
| **표시** | 절대값 + 전일/전주 대비 증감률 |
| **노출 위치** | 개요 > KPI 카드 8번 |

## 2. 전환율 KPI (퍼널 페이지용)

### CVR-01: 가입→업로드 시작 전환율

| 항목 | 값 |
|------|-----|
| **계산식** | `업로드 시작 유저 / 가입 유저 × 100` (동일 코호트 기준) |
| **건강 기준** | 40% 이상 = 양호, 25–39% = 주의, 25% 미만 = 위험 |
| **개선 레버** | 온보딩 플로우, 가입 직후 CTA, 업로드 가이드 이메일 |

### CVR-02: 업로드 시작→제출 전환율

| 항목 | 값 |
|------|-----|
| **계산식** | `업로드 제출 유저 / 업로드 시작 유저 × 100` |
| **건강 기준** | 70% 이상 = 양호, 50–69% = 주의, 50% 미만 = 위험 |
| **개선 레버** | 업로드 UX 간소화, 이어하기 리마인더, 파일 포맷 가이드 |

### CVR-03: 업로드 제출→분석 완료 전환율

| 항목 | 값 |
|------|-----|
| **계산식** | `분석 완료 건 / 업로드 제출 건 × 100` |
| **건강 기준** | 95% 이상 = 정상, 90–94% = 점검 필요, 90% 미만 = 시스템 이슈 |
| **개선 레버** | AI 파이프라인 안정성, 에러 핸들링, 재시도 로직 |

### CVR-04: 분석 완료→결제 전환율

| 항목 | 값 |
|------|-----|
| **계산식** | `결제 성공 유저 / 분석 완료 유저 × 100` |
| **건강 기준** | 25% 이상 = 양호, 15–24% = 주의, 15% 미만 = 위험 |
| **개선 레버** | 무료 프리뷰 품질, 가격 카피, 결제 유도 자동화, 사회적 증거(후기) |

### CVR-05: 결제→리포트 확인 전환율

| 항목 | 값 |
|------|-----|
| **계산식** | `리포트 확인 유저 / 결제 성공 유저 × 100` |
| **건강 기준** | 90% 이상 = 정상, 80–89% = 리마인더 필요, 80% 미만 = UX 이슈 |
| **개선 레버** | 결제 직후 리포트 자동 이동, 이메일/카카오 리포트 링크 |

### CVR-06: 리포트 확인→업체문의 전환율

| 항목 | 값 |
|------|-----|
| **계산식** | `업체문의 유저 / 리포트 확인 유저 × 100` |
| **건강 기준** | 30% 이상 = 양호, 20–29% = 주의, 20% 미만 = CTA/업체목록 개선 필요 |
| **개선 레버** | 리포트 내 업체추천 CTA, 협상 팁 콘텐츠, 업체 매칭 UX |

### CVR-07: 후기 전환율

| 항목 | 값 |
|------|-----|
| **계산식** | `후기 작성 유저 / (결제 성공 + 업체문의) 유저 × 100` |
| **건강 기준** | 10% 이상 = 양호, 5–9% = 보통, 5% 미만 = 후기 요청 강화 필요 |
| **개선 레버** | 후기 요청 자동화, 인센티브 (추천코드), 간편 후기 폼 |

## 3. 운영 KPI (개요 액션 카드용)

### OPS-01: 미결제 대기자

| 항목 | 값 |
|------|-----|
| **의미** | 분석 완료 후 24시간 이상 결제하지 않은 사용자 수 |
| **계산식** | `COUNT(users WHERE has analysis_completed AND NOT has payment_succeeded AND analysis_completed.occurred_at < NOW() - 24h)` |
| **조치** | 결제 유도 자동화 트리거, 수동 넛지 검토 |
| **심각도** | 20명 이상 = 빨강, 10–19 = 주황, 10 미만 = 기본 |

### OPS-02: 업로드 중단자

| 항목 | 값 |
|------|-----|
| **의미** | 업로드를 시작했으나 6시간 이상 제출하지 않은 사용자 수 |
| **계산식** | `COUNT(users WHERE has upload_started AND NOT has upload_submitted AND upload_started.occurred_at < NOW() - 6h)` |
| **조치** | 이어하기 링크 자동 발송 |

### OPS-03: 결제 실패 반복자

| 항목 | 값 |
|------|-----|
| **의미** | 최근 7일 내 결제 실패가 2회 이상인 사용자 수 |
| **계산식** | `COUNT(users WHERE COUNT(payment_failed, last 7d) >= 2)` |
| **조치** | 결제 수단 변경 안내, CS 연결 |

### OPS-04: 휴면 복귀 대상

| 항목 | 값 |
|------|-----|
| **의미** | 마지막 활동 후 7일 이상 경과한 가입 유저 중 결제 미완료자 |
| **계산식** | `COUNT(users WHERE last_event.occurred_at < NOW() - 7d AND NOT has payment_succeeded)` |
| **조치** | 휴면 복귀 캠페인 대상 |

## 4. 이상징후 KPI (개요 경고 카드용)

### ALERT-01: 전환율 급락

| 항목 | 값 |
|------|-----|
| **트리거** | 주요 전환율(CVR-01~06 중 하나)이 전주 대비 20%p 이상 하락 |
| **예시** | 결제 전환율이 지난주 28% → 이번주 6% = 22%p 하락 → 경고 |
| **심각도** | 20%p+ 하락 = 빨강, 10–19%p = 주황 |
| **운영자 액션** | 퍼널 페이지에서 병목 분석, 기술 이슈 점검 |

### ALERT-02: 결제 성공률 하락

| 항목 | 값 |
|------|-----|
| **계산식** | `결제 성공 / (결제 성공 + 결제 실패) × 100` |
| **트리거** | 90% 미만으로 하락 |
| **심각도** | 80% 미만 = 빨강, 80–89% = 주황 |
| **운영자 액션** | PG사 상태 확인, 결제 수단별 실패 분석 |

### ALERT-03: 업로드 실패율 상승

| 항목 | 값 |
|------|-----|
| **계산식** | `업로드 실패 이벤트 / 업로드 시작 × 100` (시스템 에러 기준) |
| **트리거** | 5% 이상 |
| **심각도** | 10%+ = 빨강, 5–9% = 주황 |
| **운영자 액션** | 서버/AI 파이프라인 점검, 파일 포맷 이슈 확인 |

## 5. 증감 표시 규칙

### 전일 대비 (일간)

```
증감값 = (오늘 - 어제) / 어제 × 100
표시: "▲ 12.5%" (green) 또는 "▼ 8.3%" (red)
어제 값이 0이면: "신규" 뱃지 표시
```

### 전주 대비 (주간)

```
증감값 = (이번주 월~일 / 지난주 월~일) × 100 - 100
표시: "전주 대비 +15.2%" 또는 "전주 대비 -5.1%"
```

### 색상 규칙

| 지표 유형 | 증가 시 | 감소 시 |
|-----------|---------|---------|
| 가입, 업로드, 결제, 매출, 문의 | green | red |
| 이탈률, 실패율 | red | green |
| 중립 지표 | blue | blue |
