## 2편 | 실패에서 배운 것

**제목 (내부용):** 대시보드 v1 실패기 - 정보 과잉에서 핵심만 남기기까지

**본문:**

> 대시보드 v1은 솔직히 실패였어요.
>
> 처음엔 "다 보여주면 좋겠지" 싶어서 차트 6개, 테이블 3개, 알림 2개를 한 화면에 넣었어요.
> 결과? 아무도 안 봤어요.
>
> 인테리어 사장님들한테 물어보니까 답은 간단했어요:
> "오늘 뭐 해야 하는지, 돈 얼마 들어오는지만 알려줘."
>
> 그래서 v2는 KPI 4개 + Quick Actions로 줄였어요.
> 정보를 빼는 게 정보를 넣는 것보다 10배 어려웠어요.
>
> 만드는 사람이 아니라 쓰는 사람 기준으로 생각하기.
> 당연한 말인데, 직접 겪어야 진짜 느껴지더라구요.

**해시태그:**
`#빌드인퍼블릭 #스타트업실패 #UX교훈 #인테리어코치 #제품개발`

**이미지 권장사항:**
- v1 대시보드 (복잡했던 화면) vs v2 대시보드 (깔끔한 화면) 나란히 비교

## 4편 | 사용자 피드백 반영

**제목 (내부용):** 인테리어 사장님이 진짜 원한 기능들

**본문:**

> "엑셀로 수금 관리하다가 300만원 빠뜨렸어요."
>
> 인테리어 사장님한테 직접 들은 말이에요.
> 이 한 마디가 캐시플로우 요약 기능을 만들게 된 이유예요.
>
> 사장님들이 요청한 것 vs 실제로 만든 것:
> -- "수금 언제 받아야 하는지 알려줘" -> 수금률 KPI + 미수금 알림
> -- "이번 주 뭐 하는지 한 눈에" -> 다가오는 일정 위젯 (오늘/이번 주)
> -- "견적서에서 바로 계약으로 넘기고 싶어" -> Quick Actions 원클릭 전환
>
> 좋은 제품은 만드는 사람이 아니라 쓰는 사람이 알려줘요.
> 사장님들 피드백이 로드맵이에요.
>
> 인테리어 현장에서 제일 불편한 업무가 뭔가요?

**해시태그:**
`#인테리어코치 #고객피드백 #빌드인퍼블릭 #프로덕트개발 #인테리어업체`

**이미지 권장사항:**
- 실제 피드백을 정리한 카드 디자인 (인용 형태, 말풍선)
- 또는 "사장님 요청 -> 구현 결과" 화살표 매핑 이미지
- 캐시플로우 섹션의 실제 UI 스크린샷 첨부
