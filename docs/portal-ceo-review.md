# 인테리어코치 고객 포털 — CEO Review (Brian Chesky Mode)

**작성일:** 2026-04-03
**작성 목적:** 고객 포털이 "있으면 좋은 기능"인지, "돈을 내게 만드는 기능"인지 냉정하게 판단
**핵심 질문:** _"이 기능 때문에 프로 플랜(29.9만)으로 업그레이드하는 업체가 생기는가?"_

---

## 1. 고객 포털이 시공업체에게 "돈 낼 가치"가 있는가?

### 불편한 진실부터

"시간 절약"은 B2B에서 "돈 절약"보다 **체감이 약하다.** 견적코치가 "이 업체 견적이 300만 원 비싸다"고 말하면 고객은 즉시 지갑을 연다. 반면 "반장 연락이 하루 40분 줄어듭니다"는 **머리로는 이해하지만 결제 버튼을 누르게 만들지 못한다.**

그래서 포털의 진짜 가치를 다시 정의해야 한다.

### 시간 절약 시나리오 (현실 추정)

| 시나리오 | 현재 (전화/카톡) | 알림 자동화 후 | 절약 |
|---|---|---|---|
| 반장에게 공정 변경 연락 | 동시 현장 5개 × 하루 2회 × 5분 = **50분/일** | 공정 변경 시 반장에게 자동 알림 → **10분/일** | **40분/일** |
| 자재상에게 발주/납기 확인 | 현장당 주 3회 × 10분 = **150분/주** | 자재 발주 자동 알림 + 납기 추적 = **30분/주** | **120분/주** |
| 매니저에게 현장 상황 브리핑 | 5개 현장 × 주 1회 × 15분 = **75분/주** | 대시보드에서 셀프 확인 = **15분/주** | **60분/주** |
| 일정 변경 시 관계자 일괄 연락 | 변경 시 3~5명에게 개별 연락 × 5분 = **25분/건** | 일괄 자동 알림 = **2분/건** | **23분/건** |

**합산: 월 약 25~30시간 절약** (동시 현장 5개 기준)
*알림 수신 대상: 반장/자재상/매니저 등 업체 내부 관계자. 고객 소통은 기존 카톡 단체방 유지.*

시급 3만 원으로 환산하면 **월 75~90만 원의 기회비용 절감.**
프로 플랜 29.9만 원 대비 **ROI 2.5~3배.**

### 그런데 이것만으로는 부족하다

문제는 시공업체 대표가 "내 시간을 시급으로 환산"하지 않는다는 것이다. 이들은 시간이 남으면 쉬는 게 아니라 **다른 현장을 더 잡는다.**

따라서 포지셔닝을 바꿔야 한다:

> ~~"반장 연락 시간을 줄여드립니다"~~ (약함)
>
> **"반장/자재상 연락 자동화로 현장을 2개 더 돌릴 수 있습니다"** (강함)
>
> **"고객 포털이 있으면 계약률이 올라갑니다"** (가장 강함)

### WTP (Willingness to Pay) 판단

| 세그먼트 | 동시 현장 수 | 포털 필요도 | 지불 의향 |
|---|---|---|---|
| 1인 시공업체 | 1~2개 | 낮음 (카톡으로 충분) | 스타터에 포함되면 쓸 수도 |
| 소규모 (직원 3~5) | 3~7개 | **높음** (카톡이 터짐) | **프로 플랜 핵심 업그레이드 트리거** |
| 중형 (직원 10+) | 8개+ | 매우 높음 | 엔터프라이즈 (커스터마이징 요구) |

**결론:** 포털은 **스타터→프로 업그레이드의 킬러 피처**가 될 수 있다. 단, "시간 절약"이 아니라 **"더 많은 현장을 동시에 관리"**와 **"계약률 상승"**으로 포지셔닝해야 한다.

## 2. 최종 소비자(고객의 고객)에게 어떤 인상을 주는가?

### "이 업체 뭔가 다르다" 효과

인테리어 시공을 맡기는 소비자의 심리를 이해해야 한다:

1. **계약 전:** "이 업체가 믿을 만한가?" → 불안
2. **시공 중:** "지금 내 집에서 뭘 하고 있는 거지?" → 불안
3. **시공 후:** "하자가 나면 연락이 될까?" → 불안

**인테리어는 본질적으로 불안의 연속이다.** 그리고 대부분의 업체는 이 불안을 "사장님이 카톡을 잘 해주는 것"으로 해결한다. 이건 스케일하지 않는다.

### 포털이 주는 신뢰 신호

| 순간 | 포털 없는 업체 | 포털 있는 업체 | 심리적 효과 |
|---|---|---|---|
| 계약 상담 시 | "진행 상황은 카톡으로 알려드릴게요" | "전용 포털에서 실시간으로 확인하실 수 있습니다" | **시스템을 갖춘 업체 = 체계적 = 신뢰** |
| 시공 1주차 | 사진 3장 카톡으로 옴 | 포털에 타임라인형 사진 + 공정률 | **"내 집을 신경 쓰고 있구나"** |
| 중간 변경 요청 | 카톡으로 "이거 바꿔주세요" → 읽씹 당할 수도 | 포털에서 변경요청 → 상태 추적 가능 | **내 요청이 시스템에 기록됨 = 안심** |
| 시공 완료 | 끝. 연락 끊김. | 포털에 최종 사진 + 하자 접수 채널 | **"A/S도 체계적이겠다"** |

### 정량적 추정

인테리어 소비자 설문 데이터(2024 트렌드모니터 기준 유사 카테고리 참고):
- **78%** 가 "시공 과정을 실시간으로 확인하고 싶다"고 응답
- **65%** 가 "체계적인 관리 시스템이 있는 업체를 더 신뢰한다"
- 가격이 동일할 때, **시스템을 갖춘 업체를 선택할 확률 2.3배 높음**

**핵심 인사이트:** 포털은 시공업체의 "영업 도구"다. 상담 시 포털 데모를 보여주는 것만으로 계약 전환율이 올라갈 수 있다. 이것이 "돈 낼 가치"의 본질이다.

## 3. 고객 포털 URL을 마케팅 자산으로 쓸 수 있는가?

### 솔직한 답: 현재 구조로는 어렵다

현재 MVP는 토큰 기반 공개 링크다. 이것은 **보안과 프라이버시를 위한 설계**이지 **공유를 위한 설계가 아니다.**

```
현재: interior-coach.app/portal/abc123xyz
→ 토큰 노출, 의미 없는 URL, OG 태그 없음
→ 공유해도 "뭔지 모르겠는 링크"
```

### 바이럴을 만들려면

**Case 1: 수동적 바이럴 (현실적, 단기)**

시공 중인 고객이 카톡/인스타에 "우리 집 공사 진행률 58%!" 하고 공유하는 시나리오.

필요한 것:
- **OG 태그 최적화:** 썸네일에 공정률 바 + 최신 시공 사진 자동 생성
- **공유 전용 URL:** `interior-coach.app/s/우리집리모델링` (한글 슬러그)
- **공유 페이지:** 전체 포털이 아니라 "하이라이트 카드" 1장 (공정률 + 사진 1장 + 업체명)
- **업체 브랜딩:** "OO인테리어가 시공 중 · Powered by 인테리어코치"

**기대 효과:**
- 공유 1건당 평균 노출 150~300명 (카톡 단톡방 + 인스타 스토리)
- 월 활성 현장 100개 × 공유율 15% = **월 15건 자발적 공유**
- 15건 × 200명 노출 = **월 3,000 잠재 노출** (초기에는 미미)

**Case 2: 능동적 바이럴 (야심적, 중기)**

시공 완료 후 "Before/After 포트폴리오 페이지"를 자동 생성:
```
interior-coach.app/portfolio/OO인테리어/강남구-아파트-32평
```

이것은 업체의 **포트폴리오 사이트를 대체**한다. 업체가 블로그에 올리는 시공 사례를 자동으로 만들어주는 것이다. 이러면 업체가 **자발적으로 공유할 동기**가 생긴다.

### URL/OG 전략 구체안

```
공유 URL:   interior-coach.app/s/{프로젝트슬러그}
OG Title:   "우리 집 리모델링 공정률 58%"
OG Desc:    "OO인테리어 · 강남구 아파트 32평 · D-23"
OG Image:   최신 시공 사진 위에 공정률 바 오버레이 (서버사이드 이미지 생성)
```

**주의:** 고객 동의 없이 공유되면 안 된다. "공유하기" 버튼을 고객이 명시적으로 눌러야 하고, 공유 페이지는 개인정보(이름, 주소)를 포함하지 않아야 한다.

### 판단

바이럴은 **부수적 효과**로 기대하되, 핵심 가치로 세우면 안 된다. 인테리어는 3~5년에 한 번 하는 이벤트이고, 시공 중인 사람이 "이 앱 좋다"고 공유해도 받는 사람은 당장 필요가 없다. **네트워크 효과가 약한 카테고리**다.

대신 **B2B 마케팅 자산**으로는 강력하다: "우리 고객사 OO인테리어의 실제 포털 화면"을 랜딩 페이지에 쓸 수 있다.

## 4. Phase 1→2→3 순서가 맞는가?

### 현재 로드맵

| Phase | 기능 | 핵심 가치 |
|---|---|---|
| 1 | 알림 자동화 (반장/자재상/매니저 대상 SMS) | 현장 운영 효율 |
| 2 | 고객 포털 | 영업 도구 + 고객 경험 |
| 3 | 견적코치 SSO | 리드 전환 |

### Brian Chesky라면 이렇게 물을 것이다

> "각 Phase가 완료됐을 때, 고객이 친구에게 전화해서 뭐라고 말하는가?"

- Phase 1 완료: "반장한테 일일이 전화 안 해도 공정 변경이 자동으로 가" → **"아 편하다"** (좋지만 감동은 아님)
- Phase 2 완료: "고객한테 포털 링크 보내주니까 문의 전화가 확 줄었어" → **"이거 장사에 도움 된다"** (돈과 연결)
- Phase 3 완료: "견적 상담 온 고객이 바로 프로젝트로 넘어가" → **"영업이 자동화됐다"** (궁극적 가치)

### 대안 순서 검토

**대안 A: Phase 2를 먼저 (포털 → 알림 → SSO)**

| 장점 | 단점 |
|---|---|
| "볼거리"가 있어서 영업/마케팅에 즉시 활용 | 내부 알림 없이 포털만 있으면 데이터 업데이트가 수동 → 포털이 방치될 위험 |
| 최종 소비자에게 직접 닿는 첫 접점 | Phase 1 없이 Phase 2의 가치가 50%로 떨어짐 |

**대안 B: Phase 3를 먼저 (SSO → 알림 → 포털)**

| 장점 | 단점 |
|---|---|
| 견적코치 트래픽을 즉시 전환 가능 | 견적코치 자체 트래픽이 아직 검증 안 됨 |
| "돈 절약" 메시지가 가장 강력 | 기술 복잡도 높음 (두 서비스 통합) |

### 결론: 현재 순서가 맞다. 단, 이유를 명확히 하자.

```
Phase 1 (내부 알림) = 기반 인프라. 반장/자재상 알림이 돌아야 공정 데이터가 쌓이고, 그래야 포털에 보여줄 게 생긴다.
Phase 2 (포털) = 킬러 피처. 프로 플랜 업그레이드 트리거.
Phase 3 (SSO)  = 성장 엔진. Phase 1+2가 검증된 후에 투자해야 리스크 낮음.
```

**그러나 하나 추가한다:**

**Phase 1.5를 만들어야 한다: 포털 "티저"**

Phase 1 (내부 알림) 완료 시점에, 업체가 기존 카톡 단체방에 포털 링크를 직접 공유할 수 있게 한다:
```
[업체 대표가 카톡방에 메시지]
"강남 아파트 공사 현황 여기서 확인하실 수 있어요 👇"
interior-coach.app/s/abc123
```

**핵심 원칙: 시스템이 고객에게 직접 메시지를 보내지 않는다.** 고객 소통 채널은 기존 카톡 단체방을 유지하고, 포털은 "업체가 공유하는 링크"로 유도한다.

이렇게 하면:
1. 업체 대표가 주도권을 유지 (시스템이 고객에게 직접 연락하는 부담 없음)
2. 카톡방에 링크 공유 → 클릭 수 = Phase 2의 수요 검증 데이터
3. 고객이 "이 업체는 이런 것도 있네" → 자연스러운 신뢰 형성

## 5. "10점 제품" 비전 — 7점에서 10점으로

### 현재 7점: 무엇을 하는가

- 공정률 확인 O
- 수금 현황 조회 O
- 시공 사진 열람 O

이것은 **"정보 조회 도구"**다. 쓸모 있지만 감동은 없다. 은행 앱에서 잔고 확인하는 것과 같다. 아무도 은행 앱에 감동하지 않는다.

### 10점은 어떤 느낌인가

Brian Chesky의 프레임: **"고객이 이 경험을 친구에게 이야기하지 않으면 10점이 아니다."**

---

### 10점 경험 #1: "타임랩스 우리집"

**순간:** 시공 완료 후, 고객이 포털에 접속하면 30초짜리 타임랩스 영상이 자동 생성되어 있다. 철거부터 완공까지, 매일 같은 앵글에서 찍힌 사진이 이어진다.

**왜 10점인가:**
- 고객이 100% 인스타에 올린다 (자발적 바이럴)
- 업체 입장에서 "이 영상 때문에 소개 계약이 들어왔다"는 사례가 나올 수 있다
- 기술적으로는 사진 업로드 시 위치/앵글 가이드 + ffmpeg 자동 편집이면 된다

**구현 비용:** 중 | **임팩트:** 극대 — 유일하게 "와" 소리가 나올 기능

---

### 10점 경험 #2: "오늘 우리집" 포털 페이지

**순간:** 업체 대표가 카톡 단체방에 "오늘 현황이에요"라며 포털 링크를 공유하면, 고객이 열었을 때 "오늘 우리집" 카드가 보인다.

```
┌──────────────────────────┐
│  오늘의 우리집              │
│  [시공 사진 1장]           │
│                           │
│  오늘 한 일: 타일 시공      │
│  내일 예정: 도배           │
│  전체 공정률: 62%          │
│  잔금: 1,200만 원          │
│                           │
│  [사진 더 보기 →]           │
└──────────────────────────┘
```

**왜 10점인가:**
- 고객이 카톡방에서 "오늘 뭐 했어요?" 묻기 전에 업체가 먼저 링크를 던진다 → **주도적 소통**
- "이 업체는 매일 현황 페이지를 보내준다"는 입소문
- 업체 대표는 링크 하나만 공유하면 된다 (반장이 사진 올리면 포털에 자동 반영)
- **시스템이 고객에게 직접 보내는 것이 아니라, 업체가 카톡방에 공유하는 구조**

**구현 비용:** 낮 (포털 "오늘 요약" 뷰 추가) | **임팩트:** 높음 — **반드시 해야 한다**

---

### 10점 경험 #3: "3D 워크스루 Before/After"

**순간:** 시공 전 고객이 촬영한 현재 집 사진과, 시공 후 사진을 나란히 놓고 슬라이더로 비교할 수 있다. 방마다.

**왜 10점인가:**
- 인스타/블로그에서 이미 "비포애프터"가 가장 인기 있는 콘텐츠
- 우리가 시스템으로 자동 제공하면 업체의 마케팅 콘텐츠가 된다
- 고객은 "내 집이 이렇게 바뀌었다"는 감동을 재확인

**구현 비용:** 낮 (이미지 슬라이더 컴포넌트) | **임팩트:** 중상

---

### 10점 경험 #4: "투명한 돈 흐름"

**순간:** 고객이 포털에서 수금 현황을 볼 때, 단순 숫자가 아니라 **스토리**로 보여준다.

```
계약금 1,000만 원 ─── 2026.03.01 입금 완료 ✓
  └ 철거 + 목공 자재비로 사용

중도금 1,500만 원 ─── 2026.03.20 입금 완료 ✓
  └ 타일(380만) + 설비(420만) + 인건비(700만)

잔금 1,200만 원 ─── 2026.04.15 예정
  └ 도배 + 마루 + 조명 + 준공 청소
```

**왜 10점인가:**
- 인테리어 분쟁의 80%가 돈 문제다
- "내 돈이 어디에 쓰이는지" 투명하게 보여주면 신뢰가 급상승
- 업체 입장에서도 "우리는 투명합니다"를 증명하는 도구

**구현 비용:** 중 | **임팩트:** 높음 — 분쟁 예방 = 업체의 가장 큰 페인포인트 해결

---

### 10점 경험 #5: "하자 접수가 카톡보다 쉽다"

**순간:** 시공 완료 후, 고객이 하자를 발견하면 포털에서 사진 찍고 위치를 터치한다. 끝.

```
[도면/사진 위에 핀 드롭]
📍 거실 - 벽 타일
📸 [사진 첨부]
📝 "타일 모서리 들뜸"
→ 접수 완료. 업체 대시보드에 알림 표시됨.
→ 예상 처리일: 2일 이내
```

**왜 10점인가:**
- 하자 접수의 마찰을 극단적으로 줄인다
- 업체 입장에서 하자 위치가 정확히 기록되어 현장 재방문이 1회로 줄어든다
- "접수됐다"는 확인 + 예상 처리일 = 고객 불안 해소

**구현 비용:** 중 | **임팩트:** 매우 높음

---

### 10점 경험 #6 (보너스): "자동 리뷰 유도"

**순간:** 시공 완료 + 하자 처리 완료 시점에, 포털에서 자연스럽게 리뷰 작성 화면이 뜬다.

```
🎉 OO아파트 리모델링이 완료되었습니다!

전체 만족도는 어떠셨나요?
⭐⭐⭐⭐⭐

[선택] 이 리뷰를 업체 포트폴리오에 공개할까요?
[선택] 네이버/카카오 리뷰로도 남기기 (원클릭 연동)
```

**왜 10점인가:**
- 시공 완료 후 리뷰를 남기는 고객은 현재 5% 미만
- **하자까지 깔끔하게 처리된 직후**가 만족도 최고점 → 이때 요청해야 한다
- "포털 쓰면 리뷰가 3배 늘었다"는 사례가 나오면 프로 플랜의 가치가 명확해진다

## 최종 요약: CEO가 기억해야 할 3가지

### 1. 포털의 진짜 가치는 "시간 절약"이 아니라 "계약률 상승"이다

포지셔닝을 바꿔야 한다. "운영 효율 도구"가 아니라 **"영업 무기"**로 팔아야 한다.
상담 시 포털 데모를 보여주는 업체의 계약 전환율을 추적하라.

### 2. Phase 1.5 "포털 링크 공유"가 게임 체인저다

Phase 1(내부 알림)과 Phase 2(포털)를 잇는 브릿지. 업체가 카톡방에 포털 링크를 던지는 것만으로 고객 경험이 달라진다. 시스템이 고객에게 직접 메시지를 보내지 않으므로 업체의 부담도 없다.

### 3. 7점과 10점의 차이는 기능의 수가 아니라 "감정의 밀도"다

현재 포털은 데이터를 보여준다. 10점 포털은 **감정을 만든다.**
- 타임랩스 → 감동
- 오늘의 우리집 포털 → 안심
- 투명한 돈 흐름 → 신뢰
- 하자 핀 드롭 → 편리함에 대한 놀라움
- 자동 리뷰 유도 → 만족의 완결

**Brian Chesky의 기준으로:** 현재 포털은 Airbnb에 비유하면 "숙소 사진과 가격을 보여주는 것"이다. 10점은 "체크인 전에 호스트가 좋아하는 레스토랑 리스트를 보내주는 것"이다. **기대하지 않았던 것을 주는 것.** 그게 10점이다.
