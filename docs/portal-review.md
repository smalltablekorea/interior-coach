# 고객 포털 마케팅/브랜딩 리뷰

> 작성일: 2026-04-03
> 리뷰 대상: 인테리어코치 고객 포털 MVP 기획 전반

---

## 1. 타겟 언어 적합성

**현재 상태 평가: 4/10**

### 문제 진단

인테리어 시공업체 대표의 평균 연령대는 40~55세. 이들의 디지털 리터러시는 "카카오톡은 능숙하지만 SaaS 용어는 외국어"에 가깝다. 현재 마케팅 자료에서 사용 중인 기술 용어들이 타겟 고객의 언어와 심각하게 괴리되어 있다.

### 즉시 제거/교체해야 할 용어

| 현재 용어 | 문제 | 대체 용어 (업체 대표 언어) |
|----------|------|------------------------|
| SaaS | 모름 | "월정액 프로그램", "온라인 관리 도구" |
| 포털 | 네이버포털 연상 | "고객 전용 페이지", "우리 고객 페이지" |
| 대시보드 | 자동차 대시보드? | "한눈에 보기", "현황판", "관리 화면" |
| 클라우드 | 모호 | "인터넷으로 어디서든" |
| 온보딩 | 외국어 | "시작 안내", "처음 설정" |
| 토큰 기반 링크 | 개발자 언어 | "고객 전용 링크", "비밀 링크" |
| API | 절대 사용 금지 | 언급 자체를 피함 |
| 워크플로우 | 모름 | "업무 흐름", "일하는 순서" |
| 실시간 동기화 | 어려움 | "바로 반영", "자동 업데이트" |
| 멀티테넌트 | 절대 노출 금지 | 내부 용어로만 사용 |

### 사용해야 할 "업체 대표의 말"

- "카톡 안 해도 고객이 알아서 확인하는 페이지"
- "공정표 공유 링크"
- "고객 문자 자동 발송"
- "수금 현황 보여주기"
- "사진 자동 공유"
- "현장 사진 앨범"

### 핵심 리스크

40~50대 업체 대표에게 "SaaS"라고 말하는 순간 "이건 나한테 해당 안 되는 거구나"라는 인식이 즉시 형성된다. 첫인상에서 이탈하면 재접근이 매우 어렵다.

### 구체적 개선 제안

1. **랜딩페이지 카피 전면 재작성**: 기술 용어를 완전히 제거하고, "반장/자재상 연락 자동화로 현장 관리 효율화"처럼 내부 운영 중심 일상 언어로 전환
2. **영상 콘텐츠(맴맴곳곳)**: 기능 시연 시 "대시보드를 열면~"이 아니라 "여기 들어가면 한눈에 보이거든요~" 식의 구어체
3. **블로그(네이버)**: "인테리어 업체 사장님, 반장 연락 자동화하는 방법" 같은 문제 중심 제목
4. **박규민 테스트 시 언어 검증**: 실제로 박규민이 자연스럽게 쓰는 표현을 채록하여 마케팅 카피에 반영

## 2. 경쟁 방어 가능성

**현재 상태 평가: 3/10**

### 집닥 시나리오 분석

집닥(시공사 매칭 플랫폼)이 유사 기능을 출시할 경우를 시뮬레이션한다.

**집닥의 강점:**
- 이미 1,000개+ 시공업체 네트워크 보유
- 소비자 트래픽 확보
- 자금력 (누적 투자 수백억)
- 기존 업체 관계

**집닥이 안 할 가능성:**
- 집닥의 수익 모델은 "매칭 수수료"이지 "업체 SaaS"가 아님
- 업체에게 SaaS를 팔면 매칭 수수료 수익 모델과 충돌
- 업체가 자립하면 집닥을 떠남 → 집닥 입장에서는 업체 종속이 유리

**하지만 집닥이 한다면:**
- 6개월 내 유사 기능 출시 가능
- 기존 업체 네트워크에 무료 배포하면 인테리어코치는 대응 불가

### 방어 수단 평가

| 방어 수단 | 현재 수준 | 현실적 평가 |
|----------|----------|------------|
| **네트워크 효과** | 없음 | B2B SaaS에서는 네트워크 효과가 약함. 업체 간 데이터 공유 인센티브가 없음 |
| **데이터 Moat** | 없음 | 프로젝트 데이터가 쌓여야 의미 있으나, 초기에는 데이터 양이 부족 |
| **전환 비용** | 낮음 | 엑셀에서 옮기는 것도 쉽지만, 나가는 것도 쉬움. 6개월 이상 사용 후 축적 데이터로 Lock-in 발생 |
| **브랜드 인지도** | 극초기 | 시장 인지도 거의 0 |
| **가격 경쟁력** | 중간 | 집닥이 무료로 풀면 가격으로 방어 불가 |

### 핵심 리스크

솔직히 현 시점에서 경쟁 방어 가능성은 거의 없다. 이것은 초기 스타트업의 본질적 한계이지 인테리어코치만의 문제가 아니다.

### 구체적 개선 제안

1. **속도가 유일한 Moat**: 타겟 세그먼트(직원 3~15명 중소업체)에서 "먼저 깊이 파고든" 것이 방어. 집닥이 "범용 기능"을 출시하더라도 중소업체 특화 워크플로우에서 차별화.

2. **데이터 Lock-in 가속화**: 프로젝트 히스토리(공정 사진, 변경 이력, 수금 기록)가 6개월 이상 쌓이면 전환 비용 급상승. 초기 무료/저가로 빠르게 데이터 축적 유도.

3. **커뮤니티 Moat 구축**: 업체 대표 간 비공개 커뮤니티 운영 → 정보 공유 + 소속감 → 이탈 방지. 박규민을 첫 앰배서더로.

4. **수직 통합**: 견적코치(B2C) → 인테리어코치(B2B)로 이어지는 소비자 트래픽 파이프라인이 유일한 구조적 방어. 이 연결고리를 빨리 완성해야 함.

5. **니치 공략**: 집닥은 "인테리어 전체"를 커버하려 함. 인테리어코치는 "주거 리모델링 3~15명 업체"에 초점 → 특화된 워크플로우/공정표/자재DB가 범용 도구보다 우위.

## 3. 콘텐츠 캘린더 현실성

**현재 상태 평가: 5/10**

### 리소스 현실 분석

| 인력 | 역할 | 주당 투입 가능 시간 (추정) | 콘텐츠 생산 용량 |
|------|------|------------------------|----------------|
| 홍학관 | 마케팅 총괄, 블로그, 전략 | 15~20시간 | 블로그 2편 + SNS 기획 |
| 김은영 | 인스타그램 운영 | 8~12시간 | 피드 3~4개 + 스토리 5~7개 |
| 정해원 | 영상 (YouTube 맴맴곳곳) | 10~15시간 | 영상 0.5~1편 (편집 포함) |

### 채널별 현실적 빈도

| 채널 | 이상적 빈도 | 현실적 빈도 | 이유 |
|------|-----------|-----------|------|
| 네이버 블로그 | 주 3회 | **주 2회** | 홍학관 혼자 기획+작성+발행. 주 3회는 품질 저하 |
| Instagram | 매일 | **주 3~4회 피드 + 매일 스토리** | 김은영 전담이면 가능하나 소재 고갈 주의 |
| Threads | 매일 | **주 3회** | 인스타 콘텐츠 리포맷으로 효율화 |
| YouTube | 주 1회 | **격주 1회** | 촬영+편집 사이클 고려 시 주 1회는 번아웃 확정 |

### 질 vs 양 트레이드오프

**현재 최적 전략: 양보다 질**

이유:
- B2B 타겟은 콘텐츠 소비량이 B2C보다 적음
- 업체 대표는 "내 문제를 해결하는 1개 글"에 반응하지, "매일 올라오는 가벼운 글"에는 무관심
- 네이버 블로그 SEO에서 "양질의 글 주 2회"가 "저품질 주 5회"보다 유리 (체류시간 기반 알고리즘)

### 번아웃 방지 전략

1. **콘텐츠 배치 시스템**: 월 1회 "콘텐츠 데이" 운영. 하루에 사진 촬영 + 블로그 3~4편 초안 + 영상 2편 촬영을 몰아서 진행
2. **콘텐츠 리사이클**: 블로그 1편 → 인스타 카드뉴스 → Threads 핵심 문장 → YouTube 숏폼. 1 소재 4 채널 활용
3. **AI 보조**: 블로그 초안은 AI로 작성 → 홍학관이 업계 경험 기반으로 수정/보완. 작성 시간 50% 절감
4. **버퍼 유지**: 항상 2주치 콘텐츠를 미리 제작해두어 긴급 상황에도 공백 없음
5. **월 1회 회고**: 성과 지표(조회수, 문의 유입) 기반으로 채널별 ROI 평가 → 저성과 채널 과감히 축소

### 핵심 리스크

3명이 4개 채널을 운영하면 "다 하는데 다 애매한" 상태가 됨. 초기에는 네이버 블로그 + 인스타그램 2개 채널에 집중하고, YouTube와 Threads는 "있으면 좋은" 수준으로 내리는 것이 현실적.

## 4. 가격 전략 시장 적합성

**현재 상태 평가: 5/10**

### 중소 업체 SaaS 지불 의향 분석

**타겟 업체 프로필:**
- 직원 3~15명
- 연매출 1~5억
- 월 영업이익 500~2,000만원
- 현재 소프트웨어 지출: 월 3~10만원 (회계 프로그램, 카카오 비즈니스 등)

**SaaS 지불 의향 벤치마크 (한국 B2B):**
- 월 5만원 이하: 50~60% 지불 의향
- 월 10만원 이하: 30~40%
- 월 15만원 이상: 15~25%
- 월 30만원 이상: 10% 미만

### 현재 가격 체계 평가

| 플랜 | 가격 | 포함 기능 | 문제점 |
|------|------|----------|--------|
| 무료 | 0원 | 기본 관리 | 적절 |
| 스타터 | 14.9만원/월 | - | **진입 장벽 높음**. 월 15만원은 "처음 써보는 도구"에 내기엔 부담 |
| 프로 | 29.9만원/월 | - | 타겟 업체 기준 매출의 0.6~3%. 가치 증명 전에는 비쌈 |
| 엔터프라이즈 | 59.9만원/월 | - | 직원 15명 이상 업체용. 타겟과 맞지 않을 수 있음 |

### 포털 과금 전략 제안

**결론: 포털은 별도 과금하지 말고 스타터 이상 플랜에 포함**

이유:
1. 포털은 "업체의 기능"이 아니라 "업체 고객의 경험". 별도 과금하면 채택률 급감
2. 포털은 Lock-in 도구. 고객이 포털에 익숙해지면 업체가 인테리어코치를 해지하기 어려움
3. "고객 포털 무료 제공"은 강력한 마케팅 메시지

**구체적 개선 제안:**

| 수정 플랜 | 가격 | 포털 포함 | 핵심 |
|----------|------|----------|------|
| 무료 | 0원 | 포털 1개 프로젝트 | 체험용. 1개 프로젝트만 포털 생성 가능 |
| 스타터 | **9.9만원/월** | 무제한 포털 + 알림 10건/월 | **진입 가격 낮춤**. 10만원 심리적 마지노선 |
| 프로 | **19.9만원/월** | 무제한 포털 + 무제한 알림 + 변경요청 | 성장 업체용 |
| 엔터프라이즈 | 별도 협의 | 전체 + 커스텀 | 대형 업체 |

핵심 변경: 스타터 14.9만 → 9.9만, 프로 29.9만 → 19.9만. "10만원 이하"와 "20만원 이하"가 중소업체의 심리적 구간이다.

### 핵심 리스크

가격을 낮추면 수익성이 떨어지지만, 현 단계는 "수익 최적화"보다 "채택률 극대화"가 우선. 100개 업체 x 10만원 > 20개 업체 x 30만원 (장기 LTV 관점).

## 5. B2B+B2C 양면 네이밍

**현재 상태 평가: 6/10**

### "인테리어코치" B2B 관점

**강점:**
- "코치"라는 단어가 "가르쳐주는/도와주는" 느낌 → 업체 대표에게 "내 업무를 도와주는 도구"로 인식 가능
- "인테리어"가 명확하게 업종을 특정

**약점:**
- "코치"가 "나보다 잘 아는 사람이 지도해주는 것" → 자존심 강한 업체 대표에게 "내가 코칭받아야 하나?"라는 반감 가능
- 40~50대 업체 대표에게 "코치"는 스포츠/피트니스 연상 → B2B SaaS와의 연결이 약함

### "인테리어코치" B2C 관점

**강점:**
- 소비자 입장에서는 "인테리어 전문가가 코칭해주는 서비스" → 견적코치와 일관된 브랜드 경험
- "코치"가 신뢰감을 줌

**약점:**
- "인테리어코치"가 소비자 서비스인지 업체 서비스인지 혼란
- 소비자가 구글 검색 시 "인테리어코치"를 치면 B2B SaaS 랜딩이 뜸 → 이탈

### 포털 네이밍

**현재**: "고객 포털" — 기술 용어, 변경 필요

**제안:**
| 후보 | B2B(업체) 인식 | B2C(소비자) 인식 | 추천도 |
|------|--------------|----------------|--------|
| "우리집 공사 현황" | 어색 | 직관적, 따뜻함 | 소비자용 OK |
| "내 공사 페이지" | 중립 | 직관적 | 가장 무난 |
| "공사 알리미" | 기능 명확 | 쉬움 | Phase 1(알림) 시기에 적합 |
| "현장 리포트" | 전문적 | 약간 딱딱 | B2B 느낌 |

**최종 제안:**
- 업체(B2B) 마케팅: "고객 전용 공사 페이지" — "포털" 대신
- 소비자(B2C) 화면: "내 공사 현황" — 소비자가 보는 페이지 타이틀
- 내부/개발: "고객 포털" — 팀 내부에서만 사용

### 핵심 리스크

"인테리어코치"가 B2B와 B2C 양쪽을 커버하려다 보면, 어느 쪽에도 강하게 어필하지 못하는 "어중간한 브랜드"가 될 수 있다. 단, 현 단계에서 이름을 바꾸는 것은 비용 대비 효과가 낮다. 이름보다 **각 타겟별 메시지를 분리**하는 것이 우선.

### 구체적 개선 제안

1. **B2B 랜딩과 B2C 랜딩 분리**: 같은 도메인이라도 `/for-business`와 `/for-homeowner`로 진입점 분리
2. **포털 URL**: `portal.interiorcoach.kr/p/{token}` 대신 `myhouse.interiorcoach.kr/{token}` — 소비자 친화적
3. **마케팅 메시지 분리**: B2B는 "반장/자재상 연락 자동화 + 현장 관리 효율화", B2C는 "우리집 공사 실시간 확인" (포털 기능)

## 6. 견적코치 → 인테리어코치 크로스셀 마찰

**현재 상태 평가: 4/10**

### 현재 크로스셀 동선

```
소비자: 견적코치에서 견적 비교
          ↓
소비자: 업체 A 선택 → 계약
          ↓
업체 A: (별도로) 인테리어코치 가입
          ↓
업체 A: 인테리어코치에서 프로젝트 생성
          ↓
소비자: 포털 링크 수신 → 공사 현황 확인
```

### 마찰 지점 분석

**마찰 1: 업체의 자발적 가입 의존**
- 소비자가 견적코치에서 업체를 선택해도, 업체가 인테리어코치에 가입하지 않으면 연결 끊김
- 현재 이 갭을 메우는 장치가 없음
- 심각도: **높음**

**마찰 2: 이름 혼란**
- "견적코치"와 "인테리어코치" — "코치" 시리즈라는 건 알겠는데, 관계가 불명확
- 소비자: "견적코치에서 견적 받았는데, 인테리어코치는 뭐지? 다른 서비스?"
- 업체: "견적코치는 소비자용인데 왜 나한테 인테리어코치를 쓰라고 하지?"
- 심각도: **중간**

**마찰 3: 계정 분리**
- 소비자의 견적코치 계정과 인테리어코치 포털 접근이 연결되지 않음
- Phase 3(SSO 연동) 전까지는 별개 시스템
- 심각도: **중간** (Phase 3에서 해결 예정이나, 그 전까지 사용자 경험 단절)

**마찰 4: 가치 인식 불일치**
- 소비자에게 견적코치의 가치는 "저렴한 견적 찾기"
- 인테리어코치 포털의 가치는 "공사 투명성"
- 가치 맥락이 달라서 자연스러운 전환이 아닌 "갑자기 다른 서비스" 느낌
- 심각도: **중간**

### SSO 통합 시 UX 이슈

1. **로그인 경험**: 견적코치에서 소셜로그인 했는데, 포털은 토큰 링크 → "왜 로그인이 다르지?"
2. **알림 채널 통합**: 견적코치 알림 vs 인테리어코치 알림이 별도 → 소비자 혼란
3. **마이페이지 통합**: 견적 내역 + 공사 현황이 한 곳에서 보여야 함 → 기술적 복잡도 높음
4. **업체 연결**: 견적코치에서 선택한 업체가 인테리어코치에도 있어야 동선이 연결됨 → 업체 가입률이 핵심 병목

### 핵심 리스크

크로스셀의 가장 큰 리스크는 "업체 가입률". 소비자가 아무리 견적코치를 사용해도, 그 업체가 인테리어코치를 쓰지 않으면 포털이 존재하지 않는다. 이 닭-달걀 문제를 해결하지 않으면 크로스셀 동선 자체가 성립하지 않는다.

### 구체적 개선 제안

1. **견적코치 내 "투명 공사 배지"**: 인테리어코치를 사용하는 업체에 견적코치 검색 결과에서 배지 부여 → 업체의 인테리어코치 가입 인센티브
2. **자동 프로젝트 생성**: 견적코치에서 계약 성사 시, 업체에게 "이 고객의 프로젝트를 인테리어코치에서 관리하시겠습니까?" 자동 제안 → 가입 마찰 감소
3. **이름 정리**: 장기적으로 "코치" 브랜드 패밀리 구축. 대외적으로는 "인테리어코치"를 통합 브랜드로, "견적"은 기능 이름으로 격하. 예: 인테리어코치 > 견적 비교 / 인테리어코치 > 공사 관리
4. **단계적 SSO**: Phase 3 이전에도 "견적코치에서 받은 견적 번호를 포털에서 입력하면 연결" 같은 수동 연결 장치를 임시로 제공
5. **통합 알림 허브**: 견적 알림 + 공사 알림을 하나의 카카오 알림톡 채널에서 발송 → 소비자에게 "하나의 서비스" 인식
