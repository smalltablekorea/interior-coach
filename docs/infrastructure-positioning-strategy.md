## A. 건설업계: Procore

### 시작점 (2003~2012)

Procore는 캘리포니아의 소규모 건설회사 대표 Tooey Courtemanche가 **자기 현장 관리용**으로 만든 도구에서 출발. 초기 제품은 단순했다:
- 현장 사진 공유
- RFI(Request for Information) 관리
- 일일 보고서(Daily Log)

**핵심 통찰:** 처음부터 "건설 ERP"를 만들지 않았다. "현장 소장이 매일 쓰는 도구 하나"에서 시작했다.

### "선택 → 필수" 전환 시점

| 시기 | 이벤트 | 의미 |
|------|--------|------|
| 2013 | 서류 관리(Document Management) 추가 | 현장의 모든 문서가 Procore 안에 들어감 |
| 2014 | 재무 관리(Financial Management) 출시 | 변경지시서, 예산 추적 진입 |
| 2016 | 하도급 관리(Subcontractor Management) | **네트워크 효과 시작** — 원청이 쓰면 하청도 써야 함 |
| 2018 | 보험/안전 규정 준수 기능 | **규제 락인** — OSHA 기록을 Procore로 관리 |
| 2021 | NYSE 상장 (시가총액 $12B) | 업계 표준으로 공식 인정 |

**전환의 결정적 계기:** 2016년 하도급 관리 도입. 원청이 Procore를 도입하면, 모든 하청업체가 접속해서 일일보고/안전점검/변경지시서를 처리해야 했다. **"내가 쓰고 싶어서"가 아니라 "원청이 쓰니까 나도 써야"**가 된 순간이다.

### 이탈을 불가능하게 만든 요소

1. **프로젝트 히스토리 데이터**: 3년치 현장 데이터, 사진, RFI가 쌓이면 이동 불가. 법적 분쟁 시 이 데이터가 증거.
2. **하청 네트워크**: 수백 개 하청업체가 온보딩. 시스템을 바꾸면 모두 재교육.
3. **맞춤 워크플로우**: 자사 프로세스에 맞게 커스터마이징한 승인 절차가 Procore 안에서만 작동.
4. **보험/규제 기록**: 안전사고 기록, OSHA 보고서가 축적.

### 인테리어코치가 배울 점

| Procore 전략 | 인테리어코치 적용 |
|-------------|------------------|
| "현장 소장의 매일 도구"에서 시작 | 대시보드를 "사장님의 아침 브리핑"으로 |
| 하청 연결로 네트워크 효과 | 반장/자재상을 앱에 연결 |
| 프로젝트 히스토리가 법적 증거 | 계약서/시공 사진/정산 기록을 "분쟁 대비 아카이브"로 |
| 규제 기록 관리 | 세무/회계 데이터 관리를 통한 규제 락인 |

## B. 식당업계: 배민 사장님앱 + 포스기

### 포스기: "없으면 장사를 못 하는" 위치에 도달한 과정

```
결제 도구 → 세금 신고 도구 → 주문 수신 도구 → 매장 운영 시스템
(편의)     (규제 강제)      (생태계 강제)    (프로세스 락인)
```

1. **1990~2000년대**: 카드 결제 단말기 (있으면 좋은 것)
2. **2005**: 현금영수증 의무화 → **규제가 강제한 "필수"**
3. **2015~**: 배달앱 주문이 포스기로 자동 수신 → 생태계 강제
4. **2020~**: 키오스크 전환 → 매장 운영 시스템

### 배민 사장님앱의 진화

| 단계 | 기능 | 락인 유형 |
|------|------|----------|
| 1 | 주문 수신/확인 | 기본 기능 |
| 2 | 메뉴 관리, 영업시간 | 운영 도구 |
| 3 | 리뷰 관리, 고객 응대 | 평판 관리 → 이탈 시 리뷰 데이터 소실 |
| 4 | 매출 통계, 지역/경쟁 분석 | **데이터 락인** |
| 5 | 배민페이, 정산, 광고 | **금융 락인** |

**결정적 락인:** 사장님이 "이번 달 매출이 얼마지?"를 배민 앱에서 확인하는 순간, 주문 도구가 **경영 도구**가 된다.

### 인테리어코치가 배울 점

| 배민/포스기 전략 | 인테리어코치 적용 |
|----------------|------------------|
| 규제(현금영수증)가 만든 강제 도입 | 세무 신고 연동 → "이거 없으면 종소세 신고가 불편" |
| 주문→매출분석으로 가치 상승 | 견적→정산→수익분석으로 가치 상승 |
| 일매출 확인이라는 일상 습관 | "오늘의 현장 현황" 확인이라는 일상 습관 |

## C. 회계: 더존/경리나라/자비스

### 외부 강제력의 역할

이것이 회계 소프트웨어의 가장 강력한 무기다:

- **부가세 신고** (분기 1회) — 안 하면 가산세
- **종합소득세 신고** (연 1회) — 안 하면 가산세
- **원천세 신고** (월 1회) — 안 하면 가산세
- **전자세금계산서 발행** — 안 하면 과태료

**사업자는 "쓰고 싶어서"가 아니라 "안 쓰면 벌금"이라서 회계 소프트웨어를 쓴다.** 이보다 강력한 락인은 없다.

### 인테리어코치가 배울 점

| 더존/자비스 전략 | 인테리어코치 적용 |
|----------------|------------------|
| 세무사에게 먼저 침투 | 인테리어 관련 세무사에게 "고객사에 추천" 유도 |
| 규제 의무가 만든 강제 사용 | 전자세금계산서 발행, 간이 장부 기능 |
| 전기 이월 데이터의 전환 비용 | 과거 현장/정산 데이터의 누적이 전환 비용 생성 |

## A. 데이터 락인 — "6개월 쌓이면 이탈 불가능"

### 시간에 따라 가치가 기하급수적으로 증가하는 데이터 (핵심 3개)

**1. 자재 단가 이력 (906개 자재)**
- 6개월: 자재별 가격 변동 트렌드 → 원가 예측 가능
- 12개월: 연간 시세 패턴 → 정확한 견적 산출의 기반
- **잃으면:** 새 견적 시 단가 감을 잃음. 마진 예측 불가.

**2. 정산 리포트 누적 (현장별 수익률)**
- 6개월: 공종별 평균 원가율 산출 → 자사 기준 생성
- 12개월: 현장 유형별 수익률 패턴 → 어떤 현장을 수주해야 이익인지 판단
- **잃으면:** 수익 구조를 다시 파악하는 데 1년 소요.

**3. 작업자 평가 데이터 (40+ 공종)**
- 6개월: 반장별 작업 품질, 일정 준수율, 단가 기록
- 12개월: 인력 배치 최적화 자산
- **잃으면:** 반장 평가를 사장님 기억에만 의존.

### 전환 비용 변화

| 시점 | 전환 비용 | 사용자 심리 |
|------|----------|-----------|
| **3개월** | ~100만원 | "엑셀로 돌아갈 수 있긴 한데... 귀찮다" |
| **6개월 (임계점)** | ~250만원 | "옮기고 싶어도 이 데이터를 어떻게 옮기지?" |
| **12개월** | ~450만원 | **"이게 없으면 사업을 어떻게 하지?"** |

### 데이터 락인 강화 기능

1. **"인사이트" 탭**: 6개월+ 데이터에만 활성화되는 고급 분석 (현장 수익 예측, 자재비 트렌드, 최적 반장 추천)
2. **"내 사업 연감"**: 12개월 시점 자동 PDF 생성 → "이 리포트를 잃고 싶지 않다"
3. **CSV 내보내기는 제공하되 분석은 앱 전용**: "데이터는 가져갈 수 있지만, 인사이트는 못 가져간다"

## B. 습관 락인 — "매일 아침 앱을 여는" 기능

### 아침 브리핑 (Morning Briefing) 설계

```
┌─────────────────────────────────────────────────┐
│  4월 3일 (목) 아침 브리핑                          │
├─────────────────────────────────────────────────┤
│                                                 │
│  오늘의 현장 (3곳)                                │
│  강남 아파트  D-12  도배 작업 예정                  │
│  판교 사무실  D-25  자재 입고 예정                  │
│  송파 빌라    D-3   최종 검수                      │
│                                                 │
│  미수금 알림                                      │
│  김OO님 중도금 500만원 — 3일 경과                   │
│  박OO님 잔금 200만원 — 오늘 예정                    │
│                                                 │
│  오늘의 할일                                      │
│  ☐ 타일 자재 발주 확인 (납기 4/5)                  │
│  ☐ 이반장님 일정 컨펌                              │
│                                                 │
│  어제 요약                                        │
│  매출 +320만  지출 -180만  순이익 +140만            │
│  vs 전주 같은 요일: +12%                           │
└─────────────────────────────────────────────────┘
```

### Push 트리거 설계

| 트리거 | 시점 | 메시지 예시 |
|--------|------|------------|
| 아침 브리핑 | 매일 08:00 | "오늘 3개 현장, 미수금 500만원" |
| 미수금 경과 | 예정일 +1일 | "김OO님 중도금 500만원 미입금 (1일 경과)" |
| 반장 일정 변경 | 변경 즉시 | "이반장님이 4/5 일정을 취소했습니다" |
| 주간 리포트 | 매주 월 09:00 | "지난주 매출 1,200만원 (전주 대비 +8%)" |

### Hook 모델 적용 (Nir Eyal)

| 단계 | 설계 |
|------|------|
| **Trigger** | Push "미수금 500만원" → 습관화 후 Push 없이도 앱 열기 |
| **Action** | 앱 열기 → 대시보드 확인 (5초, 최소 마찰) |
| **Variable Reward** | "어제 매출이 예상보다 높네!" (매일 다른 데이터) |
| **Investment** | 할일 체크, 메모 작성 → 다음 Trigger의 가치 상승 |

## C. 프로세스 락인 — "업무 흐름이 앱 안에서만 돌아가는" 구조

### End-to-End 플로우

```
견적코치(B2C) → 견적관리 → 계약관리 → 현장관리 → 시공관리
                    ↓          ↓          ↓          ↓
              자재관리(906) → 자재 발주 → 자재 입고 → 자재 사용
                    ↓                                 ↓
              작업자관리 → 반장 배정 → 작업 완료 → 반장 인건비
                    ↓                                 ↓
              지출관리 ← ─ ─ ─ 비용 발생 ─ ─ ─ ─ ─ ─ ┘
                    ↓
              정산리포트 → 세무회계 → 종소세/부가세 신고
```

### "앱 밖에서 하면 번거로운" 결정적 지점

| 순위 | 연결 | 앱 밖에서 하면 | 락인 강도 |
|------|------|-------------|----------|
| **1** | 견적→계약 자동 전환 | 항목을 수동으로 다시 입력. 빠뜨리면 분쟁 | ★★★★★ |
| **2** | 지출→정산 자동 집계 | 엑셀에서 현장별 수동 집계. 10개 넘으면 지옥 | ★★★★★ |
| **3** | 정산→세무 자동 반영 | 정산 데이터를 세무사에게 별도 전달 | ★★★★☆ |
| **4** | 자재발주→지출 자동 연결 | 발주서와 지출 장부 별도 관리. 누락 위험 | ★★★★☆ |

## D. 네트워크 락인 — "업체-반장-자재상-고객" 4자 네트워크

### 네트워크 구조

```
              ┌──────────┐
              │   고 객   │  ← 포털로 시공 확인
              └────┬─────┘
                   │
┌──────────┐  ┌────┴──────────┐  ┌──────────┐
│  자재상   │←─│ 인테리어 사장님 │─→│   반 장   │
│(알림톡)   │  │ (코어 사용자)  │  │(일정확인) │
└──────────┘  └───────────────┘  └──────────┘
```

### 실현 우선순위

| 관계 | 난이도 | 효과 | 우선순위 |
|------|--------|------|---------|
| 사장님 ↔ 반장 | 낮음 | ★★★★★ | **1순위** |
| 사장님 ↔ 고객 | 낮음 (이미 구현) | ★★★★☆ | **2순위** |
| 사장님 ↔ 자재상 | 높음 | ★★★☆☆ | 3순위 |

**치킨-에그 해결:** 사장님을 먼저 확보 → 사장님이 자기 반장을 초대하는 방식.
