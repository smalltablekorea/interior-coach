### A-1. 포지셔닝 메시지

#### 업체용 (B2B) — "왜 고객 포털을 켜야 하는가"

현재 메시지: ~~"고객 카톡 응대 80% 줄이는 프로젝트 관리"~~ → 수정 필요

| # | 후보 | 핵심 어필 |
|---|------|----------|
| 1 | **"반장/자재상 연락 자동화로 현장 관리 효율화"** | 업체 내부 운영 효율 직접 어필 |
| 2 | **"공사 끝나고 '감사합니다' 듣는 업체의 비밀"** | 감정 후크, 업체 대표 자존감 자극 |
| 3 | **"고객이 직접 확인하니까, 전화가 안 옵니다"** | 포털 기능의 직관적 인과관계 |

**최종 추천: 1번 (알림 자동화) + 3번 (포털)**을 분리 운용

- 알림 자동화 (Phase 1) 메시지: **"반장/자재상 연락 자동화로 현장 관리 효율화"**
- 고객 포털 (Phase 2) 메시지: **"고객이 직접 확인하니까, 전화가 안 옵니다"**

이유: 알림 자동화의 수신 대상은 반장/자재상/매니저 등 업체 내부 관계자이다. 고객에게 시스템이 직접 메시지를 보내지 않으므로, "카톡 응대 80% 절감"은 부정확하다. 업체 내부 운영 효율(Phase 1)과 고객 경험 개선(Phase 2 포털)을 분리해서 포지셔닝한다.

#### 고객용 (B2C) — "왜 포털을 열어봐야 하는가"

현재 메시지: "내 집 공사, 실시간으로 확인하세요"

| # | 후보 | 핵심 어필 |
|---|------|----------|
| 1 | **"우리 집 공사, 지금 어디까지 왔을까?"** | 궁금증 유발, 호기심 기반 |
| 2 | **"사진으로 보는 우리 집 변화"** | 감성적, 시각적 보상 강조 |
| 3 | **"전화 안 해도, 공사 진행이 한눈에"** | 편의성 직접 전달 |

**최종 추천: 1번** — "우리 집 공사, 지금 어디까지 왔을까?"

이유: 고객은 "실시간 확인"이라는 기능보다 "우리 집이 어떻게 되고 있는지 궁금한 감정"에 반응한다. 의문문 형태가 클릭 유도에 효과적이며, 업체가 카톡방에 포털 링크를 공유할 때 함께 보내는 메시지로 적합하다.

> **참고:** 고객 포털 유도 방식은 자동 SMS가 아니라, 업체가 기존 카톡 단체방에 포털 링크를 직접 공유하는 형태이다. 시스템이 고객에게 직접 메시지를 보내지 않는다.

### A-2. 랜딩페이지 카피

> 기존 히어로: "인테리어 업체의 모든 업무, 하나로." — 이것은 전체 SaaS 랜딩이므로 유지.
> 아래는 **고객 포털 전용 섹션** 또는 별도 랜딩 `/features/customer-portal` 용 카피.

#### 히어로 섹션

**헤드라인:**
> 고객이 직접 확인하니까, 전화가 안 옵니다.

**서브카피:**
> 공정률, 수금 현황, 시공 사진을 고객 전용 페이지로 공유하세요.
> 카톡방에 링크 하나 보내면, 고객이 직접 확인합니다.

#### 3가지 핵심 가치 제안

| 아이콘 | 타이틀 | 한 줄 설명 |
|--------|--------|-----------|
| 📊 차트 | **공정률 실시간 공유** | 공종별 진행 상황을 고객이 직접 확인. "지금 어디까지 됐어요?" 전화가 사라집니다. |
| 💰 동전 | **수금 현황 투명 공개** | 계약금/중도금/잔금 납부 상태를 고객도 함께 확인. 수금 독촉이 아닌 자연스러운 안내. |
| 📸 카메라 | **시공 사진 자동 공유** | 촬영한 현장 사진이 고객 페이지에 자동 업로드. 전/중/후 변화를 한눈에. |

#### 소셜 프루프 섹션

아직 실 사용 데이터가 부족하므로, 아래 3단계로 진행:

**Phase 1 (런칭 즉시) — 자체 데이터 활용:**
- "스몰테이블디자인그룹 자체 브랜드 3개에서 먼저 사용 중"
- "시공 완료 현장 XX건의 데이터로 설계"
- 실제 포털 스크린샷 (스몰테이블 현장 데이터)

**Phase 2 (1-2개월 후) — 초기 업체 피드백:**
- "고객 문의 전화가 확실히 줄었어요" — OO인테리어 김OO 대표
- 주간 카톡 응대 시간 비교 (Before/After 그래프)

**Phase 3 (3개월 후) — 정량 데이터:**
- "포털 도입 업체 평균 고객 문의 OO% 감소"
- "고객 만족도 평점 O.O점 (5점 만점)"
- 네이버 리뷰 전환율 수치

### A-4. 견적코치 → 인테리어코치 크로스셀 시나리오

#### 플로우 개요

```
[소비자] 견적코치에서 견적 분석
         ↓
    "이 업체에 맡기겠다" 결정
         ↓
    견적코치가 업체에 고객 정보 전달
         ↓
[업체] 인테리어코치에 프로젝트 자동 생성
         ↓
    업체가 카톡방에 포털 링크 공유
         ↓
[소비자] 포털에서 공사 현황 확인
         ↓
    시공 완료 후 리뷰 작성 유도
         ↓
    리뷰가 견적코치 업체 프로필에 반영
```

#### 구체적 UI 플로우

**Step 1 — 견적코치 (소비자 측)**
- 소비자가 견적 분석 완료 → 적정 업체 추천 리스트
- 업체 프로필에 "인테리어코치 인증 업체" 배지 표시
- "이 업체에 상담 신청" 버튼 클릭
- 간단 정보 입력: 이름, 연락처, 주소, 희망 시공 시기

**Step 2 — 인테리어코치 (업체 측)**
- 업체 대시보드 "새 상담 요청" 알림
- "견적코치에서 유입" 태그 자동 부착
- 한 클릭으로 "프로젝트 생성" → 고객 정보 자동 입력
- 계약 체결 시 포털 링크 자동 생성 → 업체가 카톡방에 공유

**Step 3 — 고객 포털 (소비자 측)**
- 업체가 카톡방에 "공사 현황 여기서 확인하세요 → [링크]" 공유
- 로그인 없이 토큰 기반 접속
- 공정률/수금/사진 실시간 확인
- 시공 완료 시 "리뷰 작성" CTA 노출

#### 업체에게 이 플로우를 소개하는 방법

**핵심 메시지:** "견적코치에서 검증된 고객이 직접 찾아옵니다"

업체 입장에서의 가치:
1. **고객 획득 비용 0원** — 견적코치에서 이미 견적을 분석하고, 적정가를 인정한 고객만 유입
2. **영업 시간 절약** — 고객 정보가 자동으로 입력되고, 프로젝트가 바로 생성됨

### B-2. 톤앤매너

#### 업체 대시보드 — 유지

- **톤:** 전문적, 데이터 중심, 효율 강조
- **테마:** 다크 (#050505 + #00C471)
- **말투:** 해요체, 간결, 숫자 중심
- **예시:** "이번 달 매출 4,500만원, 전월 대비 +12%"

#### 고객 포털 — 따뜻한 라이트 방향 강화

- **톤:** 따뜻한, 안심되는, 내 집을 잘 돌봐주고 있다는 신뢰감
- **테마:** 라이트 유지 (bg-white + emerald) — 정확한 판단
- **말투:** 해요체, 구어적, 감성적
- **핵심 원칙:** "시스템이 알려주는 것"이 아니라 "업체가 정성껏 공유하는 것"처럼 느껴져야 함

#### 구체적 카피 톤 비교

| 상황 | 시스템적 (현재) | 전문적 | 따뜻한 (추천) |
|------|---------------|--------|-------------|
| 공정 상태 변경 | 공정 상태가 변경되었습니다 | 목공 공정이 완료 처리되었습니다 | 목공 작업이 완료되었어요! 다음은 도배 작업이에요. |
| 사진 업로드 | 새 사진이 등록되었습니다 | 시공 사진 3장이 업로드되었습니다 | 오늘 현장 사진이 도착했어요. 우리 집 변화를 확인해보세요! |

### B-3. 알림 메시지 톤 설계

> **핵심 원칙: 시스템이 고객에게 직접 메시지를 보내지 않는다.**
> 자동 알림 수신 대상은 **반장/자재상/매니저** 등 업체 내부 관계자만 해당한다.
> 고객 소통은 기존 카톡 단체방을 유지하며, 포털 링크는 업체가 직접 공유한다.

#### A. 내부 관계자 자동 알림 (SMS)

**수신 대상:** 반장님, 자재상, 매니저 (업체 내부 관계자)

| 상황 | 수신자 | 메시지 |
|------|--------|--------|
| 공정 변경 | 반장님 | `[OO인테리어] 강남 아파트 - 목공 공정 시작. 4/5(토)~4/12(금) 예정. 인테리어코치에서 확인하세요.` |
| 자재 발주 | 자재상 | `[OO인테리어] 강남 아파트 - 타일 발주 요청. 납기: 4/10(수). 상세 → [링크]` |
| 일정 변경 | 반장+매니저 | `[OO인테리어] 강남 아파트 - 도배 일정이 4/15 → 4/17로 변경되었습니다. 확인 부탁드립니다.` |
| 작업 완료 보고 요청 | 반장님 | `[OO인테리어] 강남 아파트 - 오늘 작업 완료 보고 부탁드립니다. (사진 포함)` |
| 수금 완료 | 매니저 | `[OO인테리어] 강남 아파트 - 중도금 1,500만원 입금 확인됨.` |

**톤:** 전문적이고 간결하게. 반장/자재상은 하루에 여러 현장 메시지를 받으므로 핵심 정보만 전달.

#### B. 고객 포털 유도 (업체가 카톡방에서 직접)

**시스템이 보내는 게 아니라, 업체 대표가 카톡 단체방에서 공유하는 메시지 템플릿:**

| 상황 | 카톡 메시지 템플릿 (업체가 복사해서 사용) |
|------|----------------------------------------|
| 포털 최초 공유 | "OOO님, 공사 현황을 실시간으로 확인하실 수 있는 페이지를 만들었어요. 공정률, 사진, 수금 현황을 언제든 보실 수 있습니다. 👇 [링크]" |
| 주간 현황 공유 | "이번 주 현황이에요. 목공 완료, 다음 주 도배 들어갑니다. 사진도 올려놨어요! [링크]" |
| 시공 완료 | "축하드려요! 공사가 모두 완료되었어요. 최종 사진과 전체 현황을 확인해보세요. [링크]" |

**포탈 내 "카톡으로 공유" 버튼:** 대시보드에서 원클릭으로 카톡 메시지 템플릿 + 포털 URL을 복사할 수 있게 제공.

#### SMS 설계 원칙

| 원칙 | 설명 |
|------|------|
| **수신자 구분 명확** | 내부(반장/자재상) = 시스템 자동 SMS. 외부(고객) = 업체가 카톡으로 직접. |

#### 폰트 가이드

| 용도 | 한글 | 영문/숫자 | 비고 |
|------|------|----------|------|
| 대시보드 UI | Pretendard | Pretendard | 이미 적용 |

| 견적코치 | Pretendard | Pretendard | 동일 |

- 헤딩: Pretendard Bold (700)
- 본문: Pretendard Regular (400)
- 강조: Pretendard SemiBold (600)
- 포털에서는 SemiBold까지만 사용 (Bold는 업체 대시보드에서 데이터 강조용)

### B-5. 업체별 커스터마이징

#### "Powered by 인테리어코치" 전략

고객 포털은 **업체의 얼굴**이어야 한다. 고객은 "인테리어코치"라는 SaaS를 인지할 필요가 없다. "내가 계약한 OO인테리어가 이렇게 잘 해주는구나"라고 느껴야 한다.

동시에, 인테리어코치의 브랜드 노출도 필요하다 — 시공 과정을 지켜본 고객의 지인이 "이거 뭘로 하는 거야?"라고 물을 수 있는 접점.

#### 플랜별 커스터마이징 범위

| 기능 | 무료 | 스타터 (14.9만) | 프로 (29.9만) | 엔터프라이즈 (59.9만) |
|------|------|----------------|-------------|---------------------|
| 고객 포털 제공 | X | O | O | O |
| 포털 헤더 로고 | - | 인테리어코치 로고 | **업체 로고** | **업체 로고** |
| 포털 헤더 텍스트 | - | "인테리어코치" | **"OO인테리어"** | **"OO인테리어"** |
| 포털 푸터 | - | "인테리어코치 제공" | "Powered by 인테리어코치" (작게) | **완전 커스텀 가능** (로고 제거 옵션) |
| 내부 알림 발신자명 | - | [인테리어코치] | **[OO인테리어]** | **[OO인테리어]** |
| 포털 컬러 커스텀 | - | X (기본 그린) | X (기본 그린) | **O (업체 브랜드 컬러)** |
| 커스텀 도메인 | - | X | X | **O** (예: portal.oo-interior.com) |

#### 구체적 구현 방향

**스타터 플랜 포털:**
```
[헤더] IC 로고 + "인테리어코치" + "우리집 현황"
[본문] 공정률, 수금, 사진
[푸터] "인테리어코치 제공 | 인테리어 업체의 모든 업무, 하나로."
```

**프로 플랜 포털:**
```
[헤더] 업체 로고 + "OO인테리어" + "우리집 현황"
[본문] 공정률, 수금, 사진
[푸터] 작은 글씨 "Powered by 인테리어코치"
```

**엔터프라이즈 플랜 포털:**
```
[헤더] 업체 로고 + "OO인테리어" + 커스텀 명칭
[본문] 공정률, 수금, 사진 + 커스텀 컬러
[푸터] 업체 자유 설정 (인테리어코치 로고 제거 가능)
[도메인] portal.oo-interior.com
```

#### 왜 이 구조인가

- **스타터:** 인테리어코치 브랜드 노출 극대화. 포털을 본 고객의 지인/다른 업체가 "인테리어코치"를 인지하는 바이럴 채널.
- **프로:** 업체 브랜드가 주인공. 업체 대표 입장에서 "내 브랜드로 고객에게 보여줄 수 있다"가 스타터→프로 업그레이드의 핵심 동기.
- **엔터프라이즈:** 완전한 화이트라벨. 대형 업체가 자사 시스템처럼 사용.

이 구조는 업셀 동기와 직결된다:
- 스타터 업체가 "고객한테 보내는 페이지에 내 로고를 넣고 싶다" → 프로 업그레이드
- 프로 업체가 "아래 Powered by 문구도 빼고 싶다" → 엔터프라이즈 업그레이드
