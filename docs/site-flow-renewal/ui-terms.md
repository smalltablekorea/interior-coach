# 현장 중심 개편 — 화면 용어 & 도움말

> 작성일: 2026-06-21
> 범위: ① 화면에 쓸 용어 표준 ② 첫 현장 등록 온보딩 도움말 문구

## 1. 화면 용어 표준

기존 화면에서 헷갈리거나 어렵게 쓰던 단어를, 사장님이 한 번에 알아듣는 말로 정리.

### 1-1. 현장 등록 화면

| 위치 | ❌ 이전(쓰지 말 것) | ✅ 새 용어 |
|---|---|---|
| 메인 액션 버튼 | 프로젝트 생성 / Create Project | **현장 한 번에 등록** |
| 입력 그룹 1 | 기본 정보 / Basic Info | **현장 정보** |
| 입력 그룹 2 | 일정 정보 / Schedule | **시작일·기간** |
| 입력 그룹 3 | 재무 정보 / Financials | **계약 금액·대금 분할** |
| 자동 생성 영역 | Auto-generated | **자동으로 깔리는 내용** |
| 토글 | 활성화 / Enable | **켜기 / 끄기** |
| 저장 | Submit / 제출 | **저장하기** |
| 미리보기 | Preview | **미리 보기** |

### 1-2. 공정 관련

| 위치 | ❌ 이전 | ✅ 새 용어 |
|---|---|---|
| 공정 묶음 | Phase / Task / Module | **공정** |
| 공정 시작일 | Start Date | **시작일** |
| 공정 종료일 | End Date / Due | **마감일** |
| 공정 일수 | Duration | **소요일** |
| 공정 순서 | Sequence / Order | **순서** |
| 공정 진행률 | Progress | **진행률** |
| 공정 상태 | Status | **상태** |
| 병렬 작업 | Parallel / Concurrent | **같이 진행** |

**공정 상태값 라벨**
| 영문 | 새 용어 |
|---|---|
| Not Started | 예정 |
| In Progress | 진행중 |
| Completed | 완료 |
| On Hold | 보류 |
| Delayed | 지연 |

### 1-3. 대금 분할 관련

| 위치 | ❌ 이전 | ✅ 새 용어 |
|---|---|---|
| 모듈명 | Payment Schedule / Installments | **대금 분할** |
| 회차 표기 | Installment 1 / Payment #1 | **계약금 / 1차 / 2차** (라벨 자유) |
| 금액 | Amount | **금액** |
| 비율 | Ratio / Percentage | **비율 (%)** |
| 받는 날짜 | Due Date | **받는 날** |
| 받은 날짜 | Paid Date | **받은 날** |
| 받은 상태 | Status: Paid / Unpaid | **완납 / 미수** |
| 부분 결제 | Partial Payment | **일부 받음** |
| 행 추가 | Add Row / + Item | **단계 추가** |
| 행 삭제 | Remove | **삭제** |
| 비율 합계 | Total | **합계** |

**대금 분할 라벨 자동완성 (한 번 클릭으로 입력)**
```
계약금 · 착수금 · 중도금 · 잔금 · 자재비 · 기성 1차 · 기성 2차 · 기성 3차 · 하자보수보증금
```

### 1-4. 일정·캘린더 관련

| 위치 | ❌ 이전 | ✅ 새 용어 |
|---|---|---|
| 캘린더 모듈 | Schedule / Calendar | **일정** |
| 간트 차트 | Gantt Chart | **공정표** |
| 마일스톤 | Milestone | **중요 일정** |
| 카드 / 이벤트 | Event | **일정 카드** |

| 충돌 알림 | Conflict warning | **겹침 알림** |

### 1-5. 자동 생성 결과 안내 라벨

자동으로 깔린 내용 위에 붙는 작은 라벨.

| 라벨 | 의미 |
|---|---|
| 자동 생성됨 | 사장님이 직접 안 적어도 된 항목 |
| 32평 기준 | 자동 생성된 일정의 기본 가정 |
| 사장님이 조정 가능 | 자유롭게 +/- 가능 |
| 다음 현장에 적용 | 이 패턴을 기본값으로 저장 |

### 1-6. 그 외 용어 통일

| ❌ | ✅ |
|---|---|
| 유저 / User | 사장님 / 사용자 |
| 워크스페이스 활성화 | 워크스페이스 사용중 |
| 클라이언트 | 고객 (소비자 대상) / 거래처 (자재상 등) |
| 프로젝트 | 현장 |
| 카탈로그 / 스펙북 | 스펙북 (모듈명 그대로) |

| 청구서 / Invoice | 세금계산서 / 청구서 |
| 영수증 OCR | 영수증 자동 인식 |

## 2. 첫 현장 등록 온보딩 도움말 문구

처음 인테리어코치에 들어와 "현장 한 번에 등록"을 눌렀을 때, 단계마다 옆에 작게 깔리는 도움말.

### 2-1. 시작 안내 (모달)

**제목**
```
처음 현장 한 번에 등록해 보세요
```

**본문**
```
이름·평수·시작일만 넣으시면
공정 14개, 일정 캘린더, 계약, 대금 분할이 자동으로 깔립니다.

다음 현장은 더 빠르게 만들어질 거예요.
```

**CTA**: `시작하기`

### 2-2. 단계별 도움말 (인풋 옆 작은 회색 텍스트)

**[현장 이름]**
```
사장님과 고객님이 한눈에 알아볼 이름이면 됩니다.
예: "잠실 32평 김OO 사장님 댁"
```

**[주소]**
```
시·구·동까지면 충분해요. 상세주소는 나중에 입력하셔도 됩니다.
```

**[평수]**
```
공정 일수는 평수에 맞춰 자동 조정됩니다.
32평이 기본값이에요.
```

**[등급]**
```
라이트 / 스탠다드 / 프리미엄 중 골라주세요.
견적 단가가 등급에 맞춰 자동으로 설정됩니다.
```

**[시작 예정일]**
```
공정 일정이 이 날짜부터 자동으로 깔립니다.
나중에 옮기실 수 있어요.
```

**[건물 유형]**
```
아파트 / 빌라 / 단독주택 / 상가 — 공정 기본값이 조금씩 달라요.
```

### 2-3. 자동 생성 결과 화면 — 안내 박스

처음 자동 생성된 결과가 보일 때 위에 깔리는 안내.

```
🪄 사장님 대신 깔아둔 내용

- 공정 14개 (예정 일수 34일)
- 일정 캘린더 자동 표시
- 계약·납부일정 4단계 (계약금 10% / 착수금 40% / 중도금 40% / 잔금 10%)

전부 사장님이 +/- 조정하실 수 있어요.
이번에 조정하신 패턴은 다음 현장 등록할 때 기본값으로 저장됩니다.
```

### 2-4. 공정 일수 조정 영역 도움말

```
공정 일수는 32평 아파트 기준으로 깔려 있어요.
사장님 현장과 맞지 않으면 드래그로 줄이거나 늘려주세요.
```

### 2-5. 대금 분할 조정 영역 도움말

```
기본은 4단계입니다.
"단계 추가"로 6단계까지, 삭제로 2단계까지 줄이실 수 있어요.

행 이름은 자유롭게 적어주세요.
"계약금", "착수금", "기성 1차" 등 자주 쓰시는 이름은 자동으로 제안됩니다.
```

### 2-6. 마무리 페이지 (저장 직전)

```
🎉 이대로 저장하시면
- 일정 캘린더에 14개 공정이 표시되고
- 계약서 발행 가능 상태로 대금 4단계가 깔려 있고
- 자재 발주·견적·세금계산서가 이 현장 데이터를 그대로 받게 됩니다.

다음 현장은 이번에 조정하신 패턴으로 더 빠르게 시작됩니다.
```

**CTA Primary**: `저장하기`
**CTA Secondary**: `한 번 더 확인`

### 2-7. 저장 후 성공 화면

```
현장 등록 완료

이제 사장님 대시보드에서
일정·계약·자재 발주를 한 곳에서 보실 수 있어요.
```

**CTA**: `이 현장으로 이동`

## 3. 두 번째 현장부터 보일 안내

**현장 등록 첫 화면 상단**
```
이전 현장에서 사용하신 패턴(공정 일수·대금 4단계)을 기본값으로 깔아두었어요.
이번 현장 입력 더 빠르게 끝납니다.
```

## 4. 빈 상태·에러 문구 보강

### 4-1. 현장 목록이 비어있을 때

```
아직 등록한 현장이 없어요.

처음 한 번만 등록해 두면
다음 현장부턴 더 빠르게 시작하실 수 있습니다.

[ 첫 현장 한 번에 등록 ]
```

### 4-2. 평수 입력 안 했을 때

```
평수가 있어야 공정 일수가 자동으로 깔려요.
대략적인 평수만 넣어주셔도 됩니다.
```

### 4-3. 대금 비율 합계가 100% 아닐 때

```
대금 비율 합계가 {합계}%입니다. 100%가 아니면 계약서 발행 시 차이가 남을 수 있어요.
이대로 저장하실 거면 그냥 진행하셔도 됩니다.

[ 그대로 저장 ]   [ 비율 조정 ]
```

### 4-4. 단계 6개 초과 시도

```
대금은 한 현장에 6단계까지 가능합니다.
더 잘게 나누고 싶으시면 한 행을 두 줄로 나눠 적어주세요.
```

### 4-5. 시작일이 과거일 때 (경고만)

```
시작일이 과거예요.
이미 진행 중이신 현장이라면 그대로 두셔도 됩니다.
```

## 5. 카피 톤 체크 (배포 전 1분 점검)

- [ ] "사장님" 호칭 일관 사용
- [ ] "프로젝트", "마일스톤" 같은 영어 마케팅 용어 0건
- [ ] "활성화 / 비활성화" → "켜기 / 끄기"
- [ ] 자동 생성된 항목 옆에는 항상 "사장님이 조정 가능" 안내
- [ ] 경고/에러 문구는 부드럽게 (강제 차단 X, 안내 + 그대로 진행 옵션)
- [ ] 단위 표기 일관 (평·일·% — 영문 표기 금지)
