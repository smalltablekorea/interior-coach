## 1. 컬러 일관성

### Primary Green (#00C471) 사용 일관성

| 항목 | 상태 | 상세 |
|------|------|------|
| CSS 변수 정의 | **양호** | `globals.css`에서 `--green: #00C471` (다크), `--green: #00A85E` (라이트) 명확히 정의 |
| CSS 변수 사용 | **주의** | `var(--green)` 713회 사용 (59개 파일). 잘 통일되어 있으나, `#00C471` 직접 하드코딩도 8개 파일에 존재 |
| 하드코딩 위치 | **주의** | `globals.css` (정의부 정상), 마케팅 페이지 6개 (`meta-ads`, `instagram`, `naver-blog`, `youtube`, `threads`, `sms`)에서 인라인 `#00C471` 사용 |

**발견된 문제점:**
- 마케팅 섹션 페이지들에서 `#00C471`을 직접 하드코딩하는 경우가 있음. 라이트 모드 전환 시 다크용 Green이 그대로 노출될 수 있음.

**개선 제안:**
- 하드코딩된 `#00C471`을 모두 `var(--green)`으로 교체 (라이트 모드에서 `#00A85E`로 자동 전환되도록)

### 보조 컬러 체계

| 컬러 | 변수 | 상태 |
|------|------|------|
| Orange | `--orange: #FF9F43` | **양호** - 일관 사용 |
| Red | `--red: #FF6B6B` | **양호** - 일관 사용 |
| Blue | `--blue: #4A9EFF` | **양호** - 일관 사용 |
| Muted | `--muted: #888888` | **주의** - 아래 참조 |

### 다크/라이트 모드 대응

| 항목 | 상태 | 상세 |
|------|------|------|
| CSS 변수 기반 테마 전환 | **양호** | `:root` (다크) / `[data-theme="light"]` (라이트) 완비 |
| Tailwind 유틸리티 직접 사용 | **개선필요** | `text-neutral-500`, `bg-neutral-*`, `text-gray-*` 등 Tailwind 기본 색상이 12개 파일, 438회 사용됨 |
| 문제 발생 파일 | **개선필요** | 주로 `marketing/sms/page.tsx` (105회), `marketing/adlog/page.tsx` (87회), `marketing/youtube/page.tsx` (57회), `marketing/naver-blog/page.tsx` (35회) 등 마케팅 섹션에 집중 |

**발견된 문제점:**
- 마케팅 섹션 페이지들이 `text-neutral-500`, `bg-gray-900` 등 Tailwind 기본 컬러를 대량 사용. 다크/라이트 모드 전환 시 색상이 테마에 반응하지 않아 깨짐 발생 가능.
- `estimates/[id]/page.tsx`, `estimates/new/page.tsx`, `expenses/page.tsx`, `portal/[token]` 등에서도 부분적으로 사용.

**개선 제안:**
- `text-neutral-500` -> `text-[var(--muted)]`
- `bg-neutral-800` / `bg-gray-900` -> `bg-[var(--card)]` 또는 `bg-[var(--border)]`
- 마케팅 섹션 전체를 CSS 변수 기반으로 마이그레이션 (우선순위 높음)

## 2. 타이포그래피

### 한글 웹폰트 적용 여부

| 항목 | 상태 | 상세 |
|------|------|------|
| 현재 폰트 | **주의** | Geist Sans / Geist Mono (Google Fonts) - 영문 폰트만 적용 |
| Pretendard 적용 | **개선필요** | Pretendard 미적용. `Pretendard` 문자열이 `EstimateDocument.tsx`, `estimates/[id]/page.tsx`, `estimates/new/page.tsx` 3개 파일에서만 발견됨 (견적서 PDF 렌더링 전용) |
| 한글 렌더링 | **주의** | 한글은 시스템 폰트 fallback (`-apple-system, BlinkMacSystemFont, sans-serif`)으로 렌더링 |

**발견된 문제점:**
- SaaS 전체 UI가 한국어인데, 한글 전용 웹폰트(Pretendard)가 적용되지 않음
- 견적서 컴포넌트에서만 Pretendard를 사용하고 있어 일관성 부족
- Windows/Android 사용자 환경에서 '맑은 고딕'으로 fallback되어 macOS와 다른 느낌

**개선 제안:**
- `layout.tsx`에서 Pretendard를 `next/font/local` 또는 CDN으로 전역 적용
- Geist는 영문/숫자 전용으로 유지하되, 한글 기본 폰트를 Pretendard로 설정
- `font-family: 'Pretendard', 'Geist', -apple-system, sans-serif` 순서 권장

### 폰트 사이즈 체계

| 사이즈 | Tailwind 클래스 | 사용 패턴 | 상태 |
|--------|----------------|----------|------|
| 10px | `text-[10px]` | 배지, 타임스탬프 | **양호** |
| 11px | `text-[11px]` | 트렌드 라벨 | **양호** |
| xs (12px) | `text-xs` | 보조 텍스트, 날짜 | **양호** |
| sm (14px) | `text-sm` | 본문, 버튼 | **양호** |
| base (16px) | `text-base` | (거의 미사용) | **주의** |
| lg (18px) | `text-lg` | 섹션 제목, 모달 제목 | **양호** |
| xl (20px) | `text-xl` | 에러 페이지 제목 | **양호** |
| 2xl (24px) | `text-2xl` | KPI 숫자 | **양호** |

**발견된 문제점:**
- `text-base`가 거의 사용되지 않고, `text-sm`이 기본 본문 사이즈로 사용됨. 의도된 것이라면 문제없으나, 전체적으로 글씨가 작을 수 있음.
- 커스텀 사이즈 `text-[10px]`, `text-[11px]` 사용은 일관적이나 디자인 토큰으로 정의되지 않음.

### 폰트 웨이트

| 웨이트 | 용도 | 상태 |
|--------|------|------|
| `font-medium` (500) | 라벨, 네비게이션 | **양호** |
| `font-semibold` (600) | 모달 제목, 섹션 헤더 | **양호** |
| `font-bold` (700) | KPI 숫자, 로고 텍스트, CTA | **양호** |

**상태: 양호** - 웨이트 사용이 일관적임.

## 3. 톤앤매너

### 빈 상태(Empty State) 메시지

| 패턴 | 예시 | 상태 |
|------|------|------|
| `EmptyState` 컴포넌트 사용 | `title="등록된 작업자가 없습니다"` | **양호** - 공통 컴포넌트 존재 |
| 인라인 빈 상태 | `<p>배정 이력이 없습니다.</p>` | **주의** - 컴포넌트 미사용 |

**발견된 문제점:**
- `EmptyState` 공통 컴포넌트가 있으나, 일부 페이지에서는 사용하지 않고 인라인 `<p>` 태그로 빈 상태를 처리
- 빈 상태 메시지 톤이 두 가지 혼재:
  - 패턴 A: "등록된 OOO가 없습니다" (목록 전체 빈 상태)
  - 패턴 B: "OOO이/가 없습니다" (필터/검색 결과 빈 상태)
  - 이 구분은 적절하나 `description`이 있을 때도 있고 없을 때도 있어 불균일
- `EmptyState` 컴포넌트의 `bg-white/[0.04]`는 라이트 모드에서 거의 보이지 않음

**개선 제안:**
- 모든 빈 상태에 `EmptyState` 컴포넌트를 통일 사용
- `EmptyState`의 배경을 `bg-[var(--border)]`로 변경하여 라이트 모드 대응
- 빈 상태 시 항상 `description` + 액션 버튼 패턴 적용

### 에러 메시지 톤

| 위치 | 메시지 | 상태 |
|------|--------|------|
| `error.tsx` | "문제가 발생했습니다" + "페이지를 불러오는 중 오류가 발생했습니다." | **양호** |
| API 응답 | "서버 오류가 발생했습니다", "세션 확인 실패" | **주의** |
| 삭제 확인 | "정말 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다." | **양호** - 일관적 패턴 |

**발견된 문제점:**
- 사용자 대면 에러 메시지는 해요체로 일관적
- 그러나 API 에러 응답(`"세션 확인 실패"`, `"입력값 오류"`)은 명사형으로 끊김 — 사용자에게 노출될 경우 톤 불일치

**개선 제안:**
- API 에러 메시지를 사용자 친화적으로: `"세션이 만료되었어요. 다시 로그인해 주세요."` 등

### CTA 버튼 텍스트 패턴

| 패턴 | 예시 | 빈도 | 상태 |
|------|------|------|------|
| 동사형 | "다시 시도", "로그아웃" | 높음 | **양호** |
| 명사+동사 | "새 워크스페이스", "업그레이드" | 중간 | **양호** |
| 영문 혼용 | (발견 안 됨) | - | **양호** |

**상태: 양호** - CTA 텍스트가 한국어로 통일되어 있음.

## 4. 로고/워드마크

### 페이지별 로고 사용 방식

| 위치 | 표시 | 상태 |
|------|------|------|
| 사이드바 (데스크탑) | 워크스페이스명 또는 "인테리어코치" 텍스트 (Green 컬러) | **주의** |
| 사이드바 (접힘 상태) | 워크스페이스명 첫 글자 또는 "IC" | **주의** |
| 랜딩 페이지 | (별도 확인 필요) | - |
| 로그인 페이지 | (별도 확인 필요) | - |

**발견된 문제점:**
- 로고 이미지(SVG/PNG)가 사용되지 않고, 텍스트로만 브랜드명 표시
- 워크스페이스명이 설정되면 "인테리어코치" 대신 워크스페이스명이 표시되어 브랜드 노출 소실

**개선 제안:**
- 브랜드 로고 SVG/PNG 아이콘 제작 (초록색 아이콘마크)
- 사이드바 접힌 상태에서 로고 아이콘 표시
- 워크스페이스명 옆에 작은 로고 아이콘 병행 표시

### 우선 개선 3가지

1. **마케팅 섹션 컬러 마이그레이션** — `text-neutral-*` / `bg-gray-*` 438회를 CSS 변수 기반으로 교체. 다크/라이트 모드 전환 시 깨짐 해결.
2. **Pretendard 전역 적용** — `layout.tsx`에서 Pretendard 웹폰트 추가. 한국어 SaaS에서 한글 폰트 부재는 브랜딩 신뢰도에 직접 영향.
3. **브랜드 로고 아이콘 제작 + OG 이미지 설정** — 소셜 공유 시 브랜드 인지도 강화.
