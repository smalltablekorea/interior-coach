# 셀프 온보딩 패턴 리서치

> 작성일: 2026-04-09
> 핵심 질문: 대표 개입 0인 상태에서, 첫 5분 안에 "아 이거 쓸만하네"를 느끼게 하려면?

---

## 1. 해외 PLG 제품의 첫 5분 온보딩

### Linear (3분 → Aha)
1. 가입 (Google SSO, 30초)
2. 팀 이름 입력 → 워크스페이스 자동 생성
3. "첫 이슈 만들기" 프롬프트 → 이슈 하나 작성
4. 뷰 전환 (리스트 → 보드 → 타임라인) → **"오 이거 Jira보다 빠르다"**
- **Aha Moment**: 뷰 전환 속도. "이게 이렇게 빠르다고?"
- **Empty State 해결**: 첫 이슈 작성 유도 + 샘플 프로젝트 옵션

### Notion (2분 → Aha)
1. 가입 (Google SSO, 30초)
2. "용도 선택" (개인/팀/교육) → 템플릿 자동 배치
3. 빈 페이지에서 `/` 슬래시 커맨드 → **"아 이렇게 만드는 거구나"**
4. 드래그로 블록 이동 → **"레고 같네"**
- **Aha Moment**: 슬래시 커맨드. 타이핑만으로 모든 블록 삽입.
- **Empty State 해결**: 시작 템플릿 자동 배치 (Getting Started, 할 일 목록)

### Height (4분 → Aha)
1. 가입 → 프로젝트 생성
2. 태스크 붙여넣기 또는 타이핑
3. AI가 자동으로 카테고리/우선순위 분류 → **"AI가 알아서 정리해주네"**
- **Aha Moment**: AI 자동 분류. 직접 안 해도 구조화됨.
- **Empty State 해결**: "기존 프로젝트 붙여넣기" 유도

### Arc (3분 → Aha)
1. 설치 → Chrome 데이터 자동 import
2. Space 개념 소개 (개인/업무 분리)
3. Split View (화면 분할) 체험 → **"이거 크롬 아니네"**
- **Aha Moment**: Split View. "이건 크롬에서 못 하는 거다."
- **Empty State 해결**: Chrome 데이터 import로 이미 채워진 상태에서 시작

### Superhuman (30분, 고비용)
1. 온보딩 미팅 예약 (필수, 30분)
2. 전문가가 1:1로 단축키 교육
3. 이메일 받은편지함을 Superhuman으로 전환
- **Aha Moment**: `e` 키로 이메일 아카이브. "이렇게 빨라?"
- **Empty State 해결**: 기존 이메일 데이터 자동 sync. 비어있지 않음.
- **인테리어코치에 부적합**: 1:1 미팅은 스케일 안 됨. 다솜 개입 필요.

## 2. Empty State 해결 패턴 비교

| 패턴 | 사례 | 장점 | 단점 | 인테리어코치 적용 |
|------|------|------|------|---------------|
| **샘플 데이터 주입** | Notion 시작 템플릿, Figma 예제 파일 | 즉시 "채워진 느낌" | "이거 내 데이터 아닌데" 느낌 | **잠실르엘 실데이터를 샘플로** |
| **가이드 투어** | 툴팁, 코치마크, 스텝 모달 | 기능 하나씩 소개 | 스킵하면 학습 안 됨 | 현재 온보딩 모달 5단계 = 이 유형 |
| **인터랙티브 템플릿** | Linear Import, Arc Chrome import | 내 데이터로 즉시 시작 | import 대상이 있어야 함 | "현장명+주소만 입력 → 자동 구성" |
| **가짜→진짜 전환** | Figma 예제→실제 디자인, Notion 템플릿→실제 문서 | 체험 후 자연스럽게 전환 | 전환 시점이 어색할 수 있음 | **기본 권고안** |

### 기본 권고안: "가짜→진짜 전환" 패턴

잠실르엘 실제 데이터를 샘플로 보여주고 → "내 현장 등록" 전환

```
Step 1: 가입 직후
→ "잠실르엘 110동 2704호" 샘플 현장이 대시보드에 표시됨
→ 공정표, 사진, 수금 현황이 실데이터로 채워져 있음
→ "이런 식으로 내 현장을 관리할 수 있어요"

Step 2: 탐색 (1-3분)
→ 공정 클릭 → 사진 확인 → 수금 현황 → "이렇게 보이는구나"
→ 톡방 탭 클릭 → 샘플 대화 (반장/자재상과의 대화 예시)

Step 3: 전환 트리거 (3-5분)
→ "내 현장도 등록해보세요" CTA
→ 현장명 + 주소만 입력 → 나머지 자동 생성
→ 샘플 데이터는 "샘플 현장" 라벨로 유지 (삭제 가능)
```

## 3. Aha Moment 도달 시간 벤치마크

| 제품 | Aha Moment 정의 | 평균 도달 시간 | 방법 |
|------|----------------|-------------|------|
| Linear | 첫 이슈 생성 + 뷰 전환 | 3분 | 셀프 |
| Notion | 슬래시 커맨드 + 블록 이동 | 2분 | 셀프 |
| Slack | 첫 메시지 전송 + 답장 수신 | 5분 | 팀 필요 |
| Figma | 첫 프레임 + 컴포넌트 사용 | 10분 | 셀프 |
| Superhuman | 단축키로 이메일 처리 | 30분 | 1:1 미팅 |
| **인테리어코치 (목표)** | **샘플 현장 탐색 + "내 현장도 이렇게?"** | **3분** | **셀프** |

## 4. 국내 B2B SaaS 셀프 온보딩 3개

### 채널톡
- 가입 → 서비스 URL 입력 → 채팅 위젯 코드 복사 → 설치
- **Aha**: 실제 고객이 채팅으로 문의 들어옴 (1시간~1일)
- **문제점**: Aha까지 시간이 김. "아직 고객 문의가 없으면" 비어있음.

### 콴다 비즈니스
- 가입 → 학교/학원 정보 입력 → 문제집 선택 → AI 채점 체험
- **Aha**: AI가 학생 답안을 자동 채점 (2분)
- **배울 점**: "AI 채점" 데모를 즉시 체험하게 함. 실데이터 없어도 동작.

### 미리캔버스
- 가입 → 템플릿 선택 → 텍스트/이미지 수정 → 다운로드
- **Aha**: "이게 이렇게 쉽게 만들어지네" (3분)
- **배울 점**: 빈 캔버스가 아니라 **템플릿에서 시작**. Empty State 없음.

## 5. 결론: 인테리어코치 온보딩 흐름 단계별 스펙

### Step 1: 가입 직후 (0~30초)

| 요소 | 스펙 |
|------|------|
| **화면** | 환영 모달: "인테리어코치에 오신 걸 환영해요!" |
| **액션** | 업체명 입력 (필수), 대표자명 (선택) → "시작하기" |
| **트리거** | 업체명 입력 완료 → 워크스페이스 자동 생성 |
| **이탈 방지** | 입력 필드 1개만. "30초면 시작할 수 있어요" |

### Step 2: 샘플 현장 체험 (30초~2분)

| 요소 | 스펙 |
|------|------|
| **화면** | 대시보드에 "잠실 르엘 110동 2704호" 샘플 현장 표시 |
| **데이터** | 공정 13개 (철거~촬영), 시공 사진 샘플, 수금 3단계 |
| **코치마크** | "이건 실제 현장 데이터예요. 이렇게 관리할 수 있어요." (3단계 툴팁) |
| **탐색 유도** | KPI 카드 → 클릭 → 공정 상세 → 사진 → 수금 순으로 유도 |
| **성공 기준** | 공정 상세 페이지 1회 이상 진입 |
| **이탈 방지** | 샘플 데이터가 있으므로 "빈 화면" 없음 |

### Step 3: "내 현장 등록" 유도 (2~3분)

| 요소 | 스펙 |
|------|------|
| **트리거** | 샘플 현장 3개 화면 이상 탐색 시 하단에 CTA 등장 |
| **CTA** | "내 현장도 이렇게 관리해보세요" + "첫 현장 등록하기" 버튼 |
| **입력** | 현장명 + 주소 (2개 필드). 나머지(공정, 고객)는 나중에. |
| **성공 기준** | 현장 1개 등록 완료 |
| **이탈 방지** | "3분이면 등록 완료!" + 입력 도중 이탈 시 "저장됐어요" 알림 |

### Step 4: 첫 활동 (3~5분)

| 요소 | 스펙 |
|------|------|
| **화면** | 등록한 현장 상세 페이지 (빈 상태) |
| **유도** | "첫 공정을 등록해보세요" → 공종 선택 드롭다운 (철거/목공/전기...) |
| **원클릭** | 공종 선택 + 시작일만 입력 → 공정 카드 생성 |
| **성공 기준** | 공정 1개 등록 |
| **이탈 방지** | "공정 하나만 등록하면 기본 설정 완료!" |

### Step 5: Aha Moment (5분)

| 요소 | 스펙 |
|------|------|
| **순간** | 등록한 현장 + 공정이 대시보드에 표시됨. 샘플 현장과 나란히. |
| **반응** | "이걸로 카톡 대신 현장 관리할 수 있겠다" |
| **강화** | "축하해요! 첫 현장이 등록됐어요. 이제 공정 변경 시 반장님에게 자동 알림이 갑니다." |
| **다음 유도** | "반장님 연락처를 등록하면 일정 변경 알림을 보낼 수 있어요" |

### 기본 권고안 vs 대안

| | 기본 권고안 | 대안 |
|---|-----------|------|
| **샘플 데이터** | 잠실르엘 실데이터 | 가상 "예시 아파트" (실데이터 노출 꺼릴 경우) |
| **Aha 목표** | 현장+공정 등록 완료 (5분) | 대시보드 샘플 탐색만 (2분, 등록은 나중에) |
| **온보딩 방식** | 코치마크 3단계 | 5단계 스텝 모달 (현재 방식 유지) |
