## A. 해외 건설 SaaS 고객 포털

### 1. Buildertrend Client Portal

| 항목 | 상세 |
|------|------|
| **고객이 보는 것** | 프로젝트 타임라인, 일일 로그(사진+메모), 문서(계약서/변경지시서), 수금 현황(인보이스 목록), Selection(마감재 선택 상태) |
| **고객이 할 수 있는 것** | Selection 승인, 변경요청 제출, 인보이스 온라인 결제, 일일 로그 열람, 사진 갤러리 |
| **로그인** | 이메일+비밀번호 (업체가 초대) |
| **모바일** | 전용 앱 (iOS/Android) |
| **브랜딩** | 업체 로고 + 브랜드 컬러 커스터마이징 가능 (유료 플랜) |

**핵심 인사이트:** Selection(마감재 선택) 기능이 킬러. 고객이 "이 타일 vs 저 타일" 고르고 승인하면 자동으로 발주 연결. 인테리어에 직접 적용 가능.

### 2. CoConstruct Client Portal

| 항목 | 상세 |
|------|------|
| **고객이 보는 것** | 예산 현황, Selection 진행률(카테고리별 바), 일정, 변경주문 이력, 메시지 |
| **고객이 할 수 있는 것** | Selection 결정, 변경주문 요청, 메시지 발송, 예산 승인 |
| **로그인** | 이메일 초대 → 비밀번호 설정 |
| **모바일** | 반응형 웹 |
| **브랜딩** | 업체 로고 + 약관 커스터마이징 |

**핵심 인사이트:** "Selection 진행률 바"가 인상적. "바닥재 ✓ 완료 / 타일 ⏳ 결정 필요 / 조명 ○ 미시작" — 고객이 뭘 결정해야 하는지 한눈에.

### 3. Procore Client Access

| 항목 | 상세 |
|------|------|
| **고객이 보는 것** | 문서(도면, RFI, 제출물), 사진 로그, 일정(간트 뷰), 예산 요약 |
| **고객이 할 수 있는 것** | 문서 검토/승인, RFI 응답, 사진 열람 |
| **로그인** | Procore 계정 (원청이 발주처 초대) |
| **모바일** | 전용 앱 |
| **브랜딩** | 프로젝트 단위 설정 |

**핵심 인사이트:** 문서 승인 워크플로우가 핵심. 대형 건설에 특화. 인테리어에는 과잉. 다만 "승인" 개념은 중간 검수에 적용 가능.

### 3개 포털 공통 패턴

| 공통 요소 | 인테리어코치 적용 |
|---------|---------------|
| 로그인 초대 방식 | 현재 토큰 기반(로그인 불필요) 유지가 더 나음 — 인테리어 고객은 계정 만들기 싫어함 |
| 마감재 Selection | 고객이 마감재를 고르고 승인 → 발주 연결. Phase 2에 적합 |
| 사진 타임라인 | 공정별 사진 자동 분류. 현재 MVP에 이미 있음 |
| 예산/수금 투명 공개 | 현재 MVP에 있음 (수금 현황). "돈이 어디에 쓰였는지" 스토리 추가 필요 |
| 변경요청 | 고객이 변경 요청 → 업체 승인/견적 → 이력 기록. Phase 2 핵심 |

## B. 국내 리모델링 업체 고객 공유 사례

### 현실

| 방법 | 사용 비율 (추정) | 장점 | 한계 |
|------|----------------|------|------|
| **카카오톡 단톡방** | 80%+ | 사진 즉시 공유, 실시간 | 파일 만료, 검색 불가, 구조 없음 |
| **네이버 밴드** | 10% | 사진 앨범, 일정 공유 | 고객이 밴드를 안 쓰면 설치 필요 |
| **구글 시트/문서** | 5% | 공정표 공유 가능 | 실시간 업데이트 안 함, 사진 불편 |
| **전용 포털** | 1% 미만 | 체계적 | 국내 인테리어 업체용 포털 사실상 없음 |

**핵심 인사이트:** "고객 전용 페이지"를 제공하는 국내 인테리어 업체는 사실상 없음. 이것 자체가 차별점. "이 업체는 전용 페이지까지 있네" → 신뢰.

## C. 공유 URL UX 레퍼런스

### Notion 공유 페이지
- 링크만으로 접근 (로그인 불필요)
- 비밀번호 보호 옵션
- "편집 가능 / 댓글 가능 / 읽기만" 권한 구분
- OG 태그로 미리보기 지원
- **배울 점:** 로그인 마찰 제로. 링크 하나면 끝.

### Linear 공유 이슈
- 이슈 URL을 외부인에게 공유하면 읽기 전용 뷰
- 상태/담당자/코멘트가 실시간 반영
- **배울 점:** "공유한 페이지가 실시간으로 업데이트된다"는 경험. 한번 보내면 계속 최신.

### Figma 공유 링크
- 뷰어/에디터 권한 구분
- 비밀번호 보호
- 임베드 가능
- **배울 점:** 공유 링크의 OG 이미지에 실제 디자인 미리보기 → 포털 링크의 OG에 공정률 표시

### 현재 인테리어코치 포털
- 토큰 기반, 30일 유효
- 로그인 불필요 (Good)
- 라이트 모드 고정
- 공정률, 수금, 사진 조회

**개선 포인트:** OG 태그 미설정 → 카톡 공유 시 미리보기 없음. 이것만 추가해도 "와 이런 것도 있네" 효과.

## 결론: 고객 포털 MVP 화면 구성

Frontend 봇 구현 스펙 수준의 디테일.

### 화면 1: 메인 (고객이 처음 보는 것)

```
┌─────────────────────────────────────┐
│  [업체 로고] OO인테리어               │
│  ─────────────────────────────────  │
│  🏠 강남 아파트 리모델링               │
│  공사기간: 4/16 ~ 5/4               │
│                                     │
│  ┌─────────────────────────┐       │
│  │  공정률  ████████░░ 65%  │       │
│  │  D-15 (5/4 인계 예정)     │       │
│  └─────────────────────────┘       │
│                                     │
│  📸 최근 사진 (3장 미리보기)           │
│  [사진1] [사진2] [사진3]  → 더보기     │
│                                     │
│  💰 수금 현황                        │
│  계약금 ✓ / 중도금 ✓ / 잔금 ○        │
│  남은 금액: 1,200만원                │
│                                     │
│  [공정 상세 보기] [사진 전체 보기]      │
└─────────────────────────────────────┘
```

**근거:** 고객이 가장 궁금한 3가지를 한 화면에: "지금 몇 %?" + "오늘 사진" + "돈 얼마 남았지?"

### 화면 2: 공정 현황

```
┌─────────────────────────────────────┐
│  공정 현황                           │
│  ─────────────────────────────────  │
│  ✅ 철거        4/16     완료        │
│  ✅ 전기 (1차)  4/17     완료        │
│  🔄 목공        4/20     진행중 60%  │
│  ⏳ 타일        4/20~25  예정        │
│  ○ 필름        4/27~28  예정        │
│  ○ 도배        4/29~30  예정        │
│  ○ 가구        5/1      예정        │
│  ○ 입주청소    5/2      예정        │
│  ─────────────────────────────────  │
│  각 공정 탭하면 → 해당 공정 사진 표시  │
└─────────────────────────────────────┘
```

**근거:** "언제 끝나요?" 질문을 선제 해소. 각 공정의 상태가 시각적으로 명확.

### 화면 3: 사진 갤러리

```
┌─────────────────────────────────────┐
│  시공 사진                           │
│  ─────────────────────────────────  │
│  [전체] [철거] [목공] [타일] ...      │
│                                     │
│  4월 20일 (목공)                     │
│  [사진] [사진] [사진]                │
│  "천장 실링팬 보강 완료"              │
│                                     │
│  4월 17일 (전기)                     │
│  [사진] [사진]                       │
│  "분전반 회로 증설"                   │
│                                     │
│  4월 16일 (철거)                     │
│  [사진] [사진] [사진] [사진]          │
│  "가구·벽체 철거 완료"               │
└─────────────────────────────────────┘
```

**근거:** 사진이 공종별 + 날짜별로 정리됨. "오늘 뭐 했어요?" 대신 직접 확인. 카톡과의 결정적 차이 = **영구 보존 + 자동 분류**.

### 화면 4: 수금 현황

```
┌─────────────────────────────────────┐
│  수금 현황                           │
│  ─────────────────────────────────  │
│  총 계약액: 3,700만원                │
│  수금 완료: 2,500만원 (67.6%)        │
│  ████████████░░░░░░ 67.6%          │
│                                     │
│  계약금  1,000만원  3/01 ✅ 완료      │
│  중도금  1,500만원  3/20 ✅ 완료      │
│  잔  금  1,200만원  5/04 ○ 예정      │
│                                     │
│  * 잔금은 인계일(5/4)에 입금 예정      │
└─────────────────────────────────────┘
```

**근거:** "얼마 남았죠?" 질문 선제 해소. 진행률 바로 직관적. 입금 예정일 표시로 독촉 없이 안내.

### 화면 5: 변경요청 (Phase 2)

```
┌─────────────────────────────────────┐
│  변경 요청                           │
│  ─────────────────────────────────  │
│  [+ 새 요청 작성]                    │
│                                     │
│  #2 거실 타일 변경 요청     처리중    │
│  "600x600 → 800x800으로"            │
│  추가 비용: 45만원 (견적 발송됨)      │
│  4/18 접수 → 4/19 견적 → 승인 대기   │
│                                     │
│  #1 주방 후드 사양 변경     완료 ✅   │
│  "기본형 → 빌트인 교체"              │
│  추가 비용: 80만원 (승인 완료)        │
│  4/15 접수 → 4/15 승인              │
└─────────────────────────────────────┘
```

**근거:** "카톡으로 바꿔달라"고 하면 기록이 안 남음. 변경요청이 시스템에 기록되면 추가 비용 분쟁 방지. CoConstruct 변경주문 패턴.

### 기본 권고안 vs 대안

| | 기본 권고안 | 대안 |
|---|-----------|------|
| **MVP 범위** | 화면 1~4 (메인/공정/사진/수금) | 화면 1~3만 (수금 제외) |
| **근거** | 수금 투명 공개가 신뢰의 핵심 | 수금 공개를 꺼리는 업체도 있음 → 토글 옵션 |
| **변경요청** | Phase 2로 분리 | MVP에 포함하되 "카톡으로 연락" 버튼으로 대체 |
