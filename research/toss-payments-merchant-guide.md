## 1. 토스페이먼츠 심사 구조 이해

토스페이먼츠 PG 심사는 **두 단계**로 이루어진다.

### 1단계: 토스페이먼츠 자체 심사 (1~3 영업일)
- 사업자 정보 검증
- 업종 적합성 검토
- 사이트/앱 운영 현황 확인

### 2단계: 카드사 심사 (최대 10~14 영업일)
- 각 카드사(신한, 국민, 현대, 삼성 등)가 개별적으로 실사
- 심사원이 실제로 사이트에 접속하여 직접 비회원으로 결제 테스트
- 상품 페이지, 결제 흐름, 환불 정책 등을 상세 점검

### 미리오픈(Pre-open) 서비스
- 신청 후 약 6분 내 신용/체크카드 결제(일시불)가 선제공
- 최대 1개월 유지, 1개월 내 정식 계약 완료 필수
- **가상계좌, 계좌이체, 에스크로는 본 계약 완료 후에만 사용 가능**

### 심사 중 변경 금지 항목
심사 진행 중 아래 항목을 변경하면 반려 처리된다:
- 사업자 정보
- 상품 카테고리
- 비회원 결제수단 설정
- 상품 판매 상태 (품절 전환 포함)

## 2. 심사 통과 체크리스트

### 2-1. 사업자 정보 표시 (법적 필수사항)

사이트 하단 푸터에 다음 정보를 **사업자등록증과 동일하게** 표시해야 한다.

| 항목 | 필수 여부 | 비고 |
|------|-----------|------|
| 상호명 | 필수 | 사업자등록증 기재명과 동일 |
| 대표자명 | 필수 | |
| 사업자등록번호 | 필수 | |
| 사업장 주소 | 필수 | |
| 유선 전화번호 | 필수 | |
| 통신판매업 신고번호 | 권장 | 없으면 국민카드 심사 제외 (간이과세자 예외) |
| 이메일 | 권장 | |
| 개인정보 관리책임자 | 권장 | |

> **주의**: 토스페이먼츠에 신청한 정보와 사이트에 표시된 정보가 **반드시 일치**해야 한다. 불일치 시 즉시 반려.

### 2-2. 상품/서비스 페이지 요구사항

- [ ] 판매 가능한 상품 또는 서비스 **1개 이상** 노출
- [ ] 상품에 고유한 이미지 사용 (중복 이미지 금지, 이미지 오류 없음)
- [ ] **상품 금액 = 결제 금액** 일치 (할인 표시 시 원가/할인가 모두 표시)
- [ ] 품절 상태 상품은 심사 기간 중 **반드시 판매 가능 상태로 유지**
- [ ] 테스트·샘플·0원 상품은 노출하지 않음
- [ ] 빈 카테고리 및 내용 없는 빈 페이지 삭제

### 2-3. 서비스 제공 정책 표시

디지털콘텐츠/SaaS 상품의 경우 상품 페이지에 다음을 명시해야 한다:

- [ ] **서비스 제공 기간** (예: "결제 후 즉시 제공, 분석 결과는 24시간 이내 이메일 발송")
- [ ] **환불 정책** (디지털 콘텐츠 특성에 따른 환불 제한 포함)
- [ ] 서비스 제공 방식 (어떻게 전달받는지)

> 실물 상품은 배송기간/반품안내가 필요하고, 디지털 콘텐츠/무형재화는 **서비스 제공기간과 환불정책**이 필수다.

### 2-4. 결제 기능 요건

- [ ] **비회원 결제 활성화** (심사원이 비회원으로 직접 테스트함)
- [ ] 신용카드 결제 활성화
- [ ] 결제창 정상 연동 (테스트 환경 + 실제 환경 모두)
- [ ] 결제 후 주문 확인/영수증 처리 정상 작동

### 2-5. 법적 페이지 구비

- [ ] 이용약관 페이지
- [ ] 개인정보처리방침 페이지
- [ ] 환불/취소 정책 명시 (별도 페이지 또는 상품 페이지 내 표시)

### 2-6. 금액 제한 규정

카테고리별로 카드사가 결제 한도를 제한한다:

| 카테고리 | 한도 |
|----------|------|
| 충전 서비스 (크레딧/포인트 충전) | 1일 5만원 미만 |
| 게임 아이템 | 1일 5만원 미만 |
| 상품권 | 월 100만원 미만 |
| 고가 상품 (500만원 이상) | 별도 카드사 협의 필요, 심사 3~4일 추가 |
| 일반 서비스/디지털콘텐츠 | 제한 없음 (단 구체적 서비스 명시 전제) |

## 3. SaaS / 디지털콘텐츠 카테고리 요구사항

### 3-1. 업종 분류

견적코치는 다음 업종으로 분류하는 것이 적합하다:

- **소프트웨어/IT 서비스** 또는 **디지털콘텐츠**
- 구체적으로: "AI 인테리어 견적 분석 서비스" (소프트웨어 이용권)

> 상품권, 충전금, 포인트가 아닌 **서비스 이용료(Service Fee)** 형태로 등록해야 한다.

### 3-2. 서비스 제공 기간 (Service Period)

**정의**: 고객이 결제한 시점부터 서비스 제공이 완료되는 시점까지의 최대 소요 시간.

토스페이먼츠는 서비스 제공 기간을 다음과 같이 정의한다:
> "고객의 결제 시점부터 제공되는 서비스가 종료되는 시점까지의 최대 소요시간. 사전 결제로 예약하는 기간과 실제 서비스 제공 기간을 모두 포함해야 한다."

**제한 규정**:
- 서비스 제공 기간이 **1년 초과** → 결제 서비스 이용 불가
- 배송 기간이 **6주 초과** → 결제 서비스 이용 불가

**견적코치 적용 예시**:
- 잘못된 표현: "AI 분석 1회" (서비스 기간 불명확)
- 올바른 표현: "AI 인테리어 견적 분석 1회 이용권 — 결제 후 즉시 분석 시작, 최대 10분 이내 결과 제공"

### 3-3. 왜 서비스 기간 표시가 중요한가

카드사는 결제와 서비스 제공 사이의 **시간 불일치 리스크**를 심사한다. 서비스 제공 기간이 불명확하면:

1. 크라우드펀딩·예약 결제처럼 먼 미래의 서비스에 사전 결제한다고 판단할 수 있음
2. 서비스 미이행 리스크로 분류되어 고위험 업종 취급
3. 가상계좌 사용이 제한되거나 거절될 수 있음

**즉, 서비스 기간을 명시하는 것 = 카드사에게 "결제 즉시 혹은 단기간 내 이행 완료"를 증명하는 것이다.**

### 3-4. 디지털콘텐츠 환불 정책 특칙

전자상거래법상 디지털콘텐츠는 제공 개시 후 청약철회가 제한될 수 있다. 다음 내용을 상품 페이지와 이용약관에 명시해야 한다:

```
"본 서비스는 디지털 콘텐츠로, 서비스 이용이 개시된 이후에는
전자상거래 등에서의 소비자보호에 관한 법률 제17조에 따라
청약철회가 제한될 수 있습니다.
단, 서비스 미이행 시에는 전액 환불이 가능합니다."
```

## 4. 가상계좌 사용 조건

### 4-1. 가상계좌 활성화 시점

- **미리오픈 기간**: 신용/체크카드만 사용 가능
- **본 계약 완료 후**: 가상계좌, 계좌이체, 에스크로 사용 가능
- **고정 가상계좌**: 리스크 검토 및 추가 계약 필요 (별도 문의)

### 4-2. 서비스 기간과 가상계좌의 관계

가상계좌는 고객이 입금하기까지 시간이 걸리기 때문에, 카드사는 가상계좌 사용 시 **서비스 제공 기간을 더 엄격하게 검토**한다.

가상계좌를 사용하려면 다음 조건을 충족해야 한다:

| 조건 | 요건 |
|------|------|
| 서비스 제공 기간 | 반드시 명시, 1년 이내 |
| 가상계좌 입금 유효기간 | 발급 후 최대 30일 (기본 7일) |
| 입금 확인 후 서비스 제공 | 입금 완료 → 즉시 서비스 시작 명시 |

### 4-3. "6개월 규정"의 실제 의미

검색 결과에서 간혹 언급되는 "6개월 규정"은 두 가지 별도 사안이다:

1. **서비스 미이용 6개월**: 가맹점이 6개월 이상 결제 서비스를 이용하지 않으면 토스페이먼츠가 직권으로 서비스를 중지할 수 있다. (이용약관 조항)
2. **신규 사업자 수수료**: 신규 사업자는 6개월간 일반 수수료가 적용되며, 6개월 후 매출 규모에 따라 우대 수수료로 소급 정산된다.

**가상계좌 활성화 자체에 "6개월 영업 후"라는 별도 규정은 공식 문서에 명시되어 있지 않다.** 다만 고정 가상계좌(누적 입금형)는 별도 리스크 심사를 거친다.

## 5. 심사 거절 사례 분석

### 5-1. "크레딧" 용어 플래그 — 핵심 이슈

**문제**: 상품명이나 서비스 설명에 "크레딧(Credit)", "포인트", "사이버머니", "충전금" 등의 용어가 포함되면, PG사와 카드사가 해당 서비스를 **선불전자지급수단** 또는 **환금성 상품**으로 오분류할 수 있다.

**법적 배경**: 2024년 9월 15일 시행된 개정 전자금융거래법으로 선불전자지급수단 규제가 대폭 강화되었다. 이 배경에서 토스페이먼츠 포함 PG사들이 "크레딧", "포인트 충전" 형태의 상품을 더 엄격하게 심사하고 있다.

**심사 거절로 이어지는 패턴**:
- 상품명: "AI 분석 크레딧 10개" → 선불전자지급수단으로 오인
- 상품명: "분석 포인트 충전 (5,000원)" → 환금성 서비스로 분류
- 서비스 설명: "크레딧을 소모하여 AI 분석 실행" → 내부 화폐 시스템 운영으로 판단

**포인트 충전 형태 서비스의 구체적 제한**:
- 카드사 정기결제(빌링) 심사 불가
- 1일 결제 한도 5만원 미만 제한
- 고객 카드별 추가 인증 강화 조치 적용
- KCP의 경우 포인트 충전업종은 업력 2년 이상 + RM팀 별도 심사 필요

**해결 방법**: "크레딧"을 "이용권"으로 리브랜딩 (아래 6장 참조)

### 5-2. 흔한 거절 사유 목록

| 거절 사유 | 발생 원인 | 해결 방법 |
|-----------|-----------|-----------|
| 서비스 제공 기간 미표시 | 상품 설명에 기간 없음 | 상품 페이지에 "결제 후 N분 이내 제공" 명시 |
| 크레딧/포인트 선불수단 오분류 | 상품명/설명에 "크레딧" 포함 | "이용권", "서비스 이용료"로 변경 |
| 사업자 정보 불일치 | 사이트와 신청서 정보 상이 | 사업자등록증 정보와 100% 일치시킴 |
| 환불 정책 미기재 | 상품 페이지에 환불 안내 없음 | 각 상품 페이지 또는 공통 페이지에 명시 |
| 비회원 결제 비활성화 | 로그인 필수로 설정됨 | 비회원 결제 허용 또는 심사용 계정 제공 |
| 품절 상품만 노출 | 실제 구매 불가 상품만 등록 | 판매 가능한 상품 1개 이상 유지 |
| 중복/저품질 상품 이미지 | 모든 상품에 동일 이미지 사용 | 상품별 고유 이미지 사용 |
| 통신판매업 신고번호 없음 | 신고 미완료 | 통신판매업 신고 후 번호 입력 (국민카드 필수) |
| 환금성 상품 미신고 | 사전 승인 없이 환금성 상품 판매 | 판매하지 않으면 "환금성 상품 없음" 명시 |
| 테스트 상품 노출 | "테스트", "sample" 상품이 심사 중 노출됨 | 심사 전 제거 또는 비공개 처리 |

### 5-3. 고위험 업종 분류 주의

PG사와 카드사가 입점을 제한하거나 별도 심사를 요구하는 업종:

**완전 금지** (어떤 PG도 불가):
- 암호화폐 거래, 도박, 대출, 성인 콘텐츠, 경매

**조건부 허용** (별도 서류/심사 필요):
- 포인트/크레딧 충전 서비스
- 게임 아이템
- 중고품 거래 플랫폼
- 컨설팅 (국가 전문자격증 소지자에 한함)
- 주류, 의약품 (관련 허가증 필요)

**견적코치는 고위험 업종이 아니다.** 단, 상품명과 서비스 설명에서 크레딧/포인트 용어를 제거하면 일반 소프트웨어 서비스로 분류되어 심사가 훨씬 수월해진다.

### 5-4. 재심사 신청 방법

심사 거절(미비 사항 통보) 후 재신청 절차:

1. 토스페이먼츠로부터 **카카오 알림톡** "미비 사항을 보완해주세요" 수신
2. 알림톡에서 **[미비 사항 보완하기]** 클릭
3. 지적된 항목 수정 완료
4. 알림톡 또는 토스페이먼츠 관리자 페이지에서 **재심사 요청**
5. **1~3 영업일** 대기

> 재심사 시에도 카드사 심사는 새로 시작된다. 보완 후 총 7~14일이 추가 소요될 수 있다.

## 6. 신규 결제 상품 추가 시 사전 점검 항목

### 6-1. 상품명 작성 원칙

**피해야 할 용어**:

| 위험 용어 | 이유 | 대체 용어 |
|-----------|------|-----------|
| 크레딧 (Credit) | 선불전자지급수단 오인 | 이용권, 서비스권 |
| 포인트 충전 | 환금성 서비스 분류 | 서비스 이용료 |
| 사이버머니 | 전자화폐로 분류 가능 | (사용 금지) |
| 코인 (서비스 내 화폐) | 가상화폐/환금성 오인 | (사용 금지) |
| 충전금 | 선불수단 직접 명시 | (사용 금지) |
| 무제한 | 서비스 기간 불명확 | "1개월 이용권" 등으로 기간 명시 |
| 테스트 / 샘플 | 실제 판매 의지 의심 | (심사 중 비공개 처리) |

**권장 상품명 형식**:

```
[서비스명] [단위 또는 횟수] 이용권 — [서비스 제공 기간/방식]

예시:
- "AI 인테리어 견적 분석 1회 이용권 — 결제 후 즉시 분석 시작"
- "견적코치 AI 분석 서비스 (1건) — 분석 완료까지 최대 10분"
- "인테리어 견적 AI 검토 서비스 단건"
```

### 6-2. 상품 페이지 필수 기재 항목 (디지털콘텐츠/SaaS)

```markdown
## 서비스 내용
[구체적인 서비스 설명: 무엇을 분석하고, 어떤 결과물을 받는지]

## 서비스 제공 방식
결제 완료 후 즉시 AI 분석이 시작되며, 분석 결과는 [10분 이내 / 즉시] 
[화면에 표시 / 이메일로 발송]됩니다.

## 서비스 제공 기간
결제 완료 후 즉시 제공. 서비스 제공 기간: 1회 (즉시 완료)

## 환불 정책
- 서비스 이용 전 (분석 시작 전): 전액 환불 가능
- 서비스 이용 후 (분석 완료 후): 디지털 콘텐츠 특성상 환불 불가
  (전자상거래 등에서의 소비자보호에 관한 법률 제17조 적용)
- 서비스 오류 또는 미이행 시: 전액 환불 보장

## 문의
고객센터: [이메일 또는 전화번호]
```

### 6-3. 사이트 필수 법적 페이지 체크리스트

심사 전 반드시 구비해야 할 페이지:

- [ ] **이용약관** (`/terms`)
  - 서비스 이용 조건
  - 결제 및 환불 조건
  - 금지 행위
  
- [ ] **개인정보처리방침** (`/privacy`)
  - 수집 항목, 수집 목적
  - 보유 기간
  - 제3자 제공 여부
  - 개인정보 관리책임자 정보

- [ ] **환불정책** (이용약관 내 또는 별도 페이지)
  - 디지털 콘텐츠 특성에 따른 환불 제한 명시

### 6-4. 상품 등록 전 최종 체크리스트

```
[ ] 상품명에 "크레딧", "포인트 충전", "코인" 등 위험 용어 없음
[ ] 상품 페이지에 서비스 제공 기간 명시 (기간 또는 "즉시 제공" 명확히)
[ ] 상품 페이지에 환불 정책 명시
[ ] 상품 금액 = 결제 금액 (숨겨진 비용 없음)
[ ] 상품 이미지 고유하게 준비됨
[ ] 비회원 결제 가능 여부 확인
[ ] 테스트 환경과 실제 환경 모두 결제창 연동 완료
[ ] 사이트 하단 푸터에 법적 필수 정보 표시
[ ] 사업자등록증 정보와 사이트 정보 100% 일치
[ ] 통신판매업 신고번호 입력 (국민카드 심사 포함 시 필수)
[ ] 이용약관 및 개인정보처리방침 페이지 정상 접근 가능
```

### 6-5. 포인트/크레딧 시스템이 필요한 경우 합법적 운영 방법

서비스 내부에서 크레딧 개념이 사용자 경험상 필요하다면, PG 결제 레이어와 내부 크레딧 레이어를 **분리**해야 한다.

**권장 구조**:
```
[PG 결제 레이어]
결제 상품: "AI 견적 분석 이용권 1회" (명확한 서비스)
↓ 결제 완료
[내부 서비스 레이어]
시스템 내부에서 "분석 1회 차감" 처리
(사용자에게 "크레딧"으로 보이더라도, 결제 상품명은 이용권)
```

**핵심**: 토스페이먼츠에 등록되는 **결제 상품명**에는 "이용권" 또는 "서비스 이용료"를 사용하고, 사이트 내부 UX에서 "크레딧" 개념을 쓰더라도 **결제창에 노출되는 상품명에는 사용하지 않는다.**

만약 선충전 크레딧 시스템(선불 충전 → 나중에 소모)을 운영하려면:
1. 포트원/토스페이먼츠 담당자와 **사전 협의 필수**
2. 개정 전자금융거래법(2024.09.15 시행)상 선불전자지급수단 등록 요건 검토
3. 발행잔액 30억원 미만, 연간 총발행잔액 500억원 미만이면 등록 면제 대상일 수 있으나 법률 전문가 확인 필요

## 7. 견적코치 적용 액션 플랜

### 현재 문제 요약

토스페이먼츠 심사에서 "크레딧" 용어가 선불전자지급수단으로 플래그되어 거절된 상황.

### 즉시 조치 항목 (재심사 전)

**1. 결제 상품명 변경**
- Before: "AI 분석 크레딧" 또는 "분석 크레딧 1개"
- After: "AI 인테리어 견적 분석 이용권 (1회)"

**2. 상품 페이지 서비스 기간 명시**
```
서비스 제공 기간: 결제 완료 후 즉시 분석 시작, 최대 10분 이내 결과 제공
```

**3. 환불 정책 상품 페이지에 추가**
```
환불 정책: 분석 시작 전 100% 환불 가능. 분석 완료 후 디지털 콘텐츠 특성상 환불 불가.
```

**4. 사이트 푸터 사업자 정보 점검**
- 사업자등록증 정보와 100% 일치 여부 확인
- 통신판매업 신고번호 입력

**5. 비회원 결제 활성화 확인**

### 재심사 신청

위 항목 수정 완료 후 → 카카오 알림톡 [미비 사항 보완하기] 클릭 → 재심사 요청

### 중기 개선 항목

- 내부 UX에서 "크레딧" 표현을 "이용권" 또는 "분석권"으로 리브랜딩 검토
- 이용약관에 디지털 콘텐츠 환불 불가 조항 명시
- 개인정보처리방침 최신화
