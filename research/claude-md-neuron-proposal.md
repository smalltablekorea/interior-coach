## 추가할 뉴런 (CLAUDE.md에 삽입)

```markdown
## PG 심사 & 결제 용어 가이드라인

### 금지어 (PG 심사 반려 트리거)
다음 용어는 코드, UI, API 경로, 상품명, 약관에서 **절대 사용 금지**:
- 크레딧 / credit / credits
- 포인트 / point (결제 맥락에서)
- 토큰 / token (결제 맥락에서. OAuth 토큰은 예외)
- 코인 / coin
- 캐시 / cash
- 충전 / charge / recharge / top-up
- 잔액 / balance (결제 맥락에서)
- 환전

**이유**: 전자금융거래법상 선불전자지급수단으로 의심받아 PG 심사 반려됨 (2026-04 토스페이먼츠 실제 반려 경험)

### 권장어
- 분석권 / analysis pass
- 이용권 / usage pass
- 1회권 / 3회권 / single pass / multi-pass
- 서비스 이용 횟수

### 결제 상품 네이밍 규칙
- 토스에 전송하는 `orderName`은 반드시 "서비스명 + 구체적 서비스 설명 + 횟수" 형태
  - 예: "견적코치 AI 분석 이용권 (1회)"
  - 금지: "크레딧 10개", "포인트 충전"

### DB/API 네이밍
- 테이블명: `analysis_passes` (not `analysis_credits`)
- 컬럼명: `total_passes`, `used_passes`, `pass_used`
- API 경로: `/api/analysis-passes/*` (not `/api/credits/*`)

### 서비스 제공 기간
- 분석권 유효기간: **180일** (가상계좌 6개월 규정 준수)
- 구독: 월간(1개월) / 연간(1년)
- 약관·환불정책에 명시 필수

### 홈페이지 필수 표시 사항 (전자상거래법 제13조)
신규 페이지 생성 또는 Footer 수정 시 반드시 포함:
1. 상호 (스몰테이블)
2. 대표자명 (배다솜)
3. 사업자등록번호 (511-27-58367)
4. 통신판매업 신고번호 (2026-인천연수구-0926)
5. 주소
6. 전화번호 / 이메일
7. 개인정보보호책임자
8. 호스팅 사업자 (Vercel Inc.)
9. 사업자정보 확인 링크 (공정거래위원회)
10. 이용약관 / 환불정책 / 개인정보처리방침 링크

### 신규 유료 기능 추가 시 체크리스트
1. [ ] 상품명에 금지어 없는지 확인
2. [ ] 서비스 제공 기간 명시 (180일 이하 or 구독 기간)
3. [ ] 환불 정책 업데이트
4. [ ] 이용약관 업데이트
5. [ ] 토스 `orderName` 포맷 확인
6. [ ] 청약철회 예외 조항 확인 (디지털 콘텐츠)
```

---

```markdown
결제/과금 관련 코드, UI, API, 약관에서 "크레딧", "포인트", "토큰", "충전" 등 선불전자지급수단을 연상시키는 용어 사용 금지.

**Why:** 2026-04 토스페이먼츠 가맹점 심사에서 "크레딧" 표현 때문에 선불충전(전자금융업) 의심으로 반려됨. 전자금융거래법 제2조 선불전자지급수단 정의에 해당하지 않더라도, PG 심사에서는 명칭 자체가 트리거가 됨.

**How to apply:** 
- 새로운 유료 기능 추가 시 상품명·DB명·API명 모두 "이용권/분석권/pass" 계열로 네이밍
- DB 컬럼명도 `credits` 대신 `passes` 사용
- 홈페이지 Footer 수정 시 전자상거래법 필수 표시 사항 10개 항목 체크
```
